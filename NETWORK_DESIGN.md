## 1. 设计红线（任何升级不得触碰）

**信息流向只许 physics → network，不许 network → style。**

网络的职责边界：修复"从估计深度出发的分层渲染"在**深度边界**的瑕疵
（渗色/硬边/半透明圈）。焦外风格——盘的形状、亮边、猫眼、旋焦、色差——
**全部由物理渲染器产出**，网络只能"知道"它们（作为条件），不能"画"它们。

误差图融合公式不变：

```
B = m · B_neural + (1−m) · B_phys      （m 集中在深度边界，平坦区 m≈0）
```

**为什么是红线**（一旦破坏，项目三个支柱全塌）：
1. **可控性证明**（RQ1）：风格若部分来自网络，解耦签名矩阵就度量不清"系数→风格"；
2. **镜头指纹**（RQ3）：对系数梯度下降反求的前提是风格完全由可微物理承载；
3. **差异化定位**：与扩散/流匹配方法的本质区别就是"风格可解释"，network→style
   等于自废武功去拼别人的主场。

**实操检查清单**——任何网络改动前问三个问题：
- [ ] 新输入/条件是不是**物理可计算量**（来自渲染器/深度/控制向量）？
- [ ] 网络输出的活动范围是否仍被 m 限制在边界？
- [ ] 去掉细化网，物理渲染是否仍能独立产出全部风格效果？

## 2. 现状 v1（baseline 存档）

结构（实现：[refine/network.py](refine/network.py)，684,035 参数 = 2M 上限的 34%）：

```
输入 guide = [全焦图 3 ‖ 扰动视差 1 ‖ B_phys 3 ‖ CoC 提示 r̃ 1] = 8ch
ctrl_vec 13 维 → CondEncoder(2 层 MLP, 64d) → 各块 FiLM (γ,β)
ARNet：guide↓2 → PixelUnshuffle(2) → 96ch × 3 FiLM 残差块 → 粗散景残差 + 误差图 logit
IUNet：[guide 全分 ‖ 粗散景↑] → 48ch × 2 FiLM 残差块 → B_neural
融合：B = m·B_neural + (1−m)·B_phys
```

已验证（PROJECT_STATUS §6）：
- 10k 步（占位背景池）：边界 L1 较物理基线 **−27%**；
- 50k 步 COCO 正式训练进行中（9k 步时 −17%，验证集更难、仍在下降）；
- 滑窗端到端推理（D20）：收益不因滑窗损失。

**v1 的四个已知缺陷**（升级路线的靶子）：
1. (H_field, azimuth) 是**整块一个标量**走 FiLM → 训练时假装整块在画幅同一点，
   推理被迫滑窗（D20），且窗内 H 恒定近似在**画幅边角误差最大**——而那里恰是
   猫眼/旋焦最强处；
2. 网络对**局部 PSF 形状盲**：只知系数数值，不知"此处的核长什么样"，
   不同像差风格下的边界修复策略无法分化；
3. plain conv 残差块的参数效率已被 NAFNet 系明显超越（同预算性能差距实测约 0.3–0.5dB 级）；
4. 合成管线的 **GT alpha 监督信号完全未用**（白白扔掉的免费监督）。

## 3. 文献坐标与空位（2026-06 调研结论）

**条件注入机制的演进线**（散景渲染 + 相邻领域）：

```
镜头 one-hot + 光圈数值嵌入          BokehOrNot / NAFBET (NTIRE23)
  → f 值 → 注意力掩码宽度            Bokehlicious (ICCV25)
  → 文本/控制 cross-attention        BokehFlow / Generative Refocusing (2025)
  → Latent PSF 先验（VQVAE 学码本）  OmniLens++ (2025.11，像差【校正】域)
  → ??? 像差系数 + PSF 形状条件      ← 空位，本项目的位置
```

**关键空位**：没有任何散景渲染工作让网络以**实际 PSF 形状/像差系数**为条件——
最接近的 OmniLens++ 在反问题（去像差）侧，且因为"盲"设定只能用 VQVAE 学 PSF
潜表征；我们是**非盲**的（PSF 自己渲染的），可以直接解析提取，更干净。

**撞车检查**（2025.11–2026.06 新作）：BokehFlow（流匹配、焦点+强度控制）、
Generative Refocusing（DeblurNet+BokehNet、EXIF 条件）、BokehDepth——全部停留在
"虚多少/虚哪里"层面，无人做像差系数级控制。**市场佐证**：Voigtländer 2026.02 发布
带球差控制环的实体镜头、Lomography Nour Triplet 的 SA 旋钮——"球差可调"有真实
摄影需求，可写进论文 motivation。

## 4. P1：条件机制升级（创新点核心，最优先）

### 4.1 P1a — 空间物理条件图

**改什么**：guide 从 8ch 扩到 11ch，新增三个**逐像素物理图**：

```
H_map          [1,H,W]  每像素归一化像高（render_tiled 的 T(H) 插值已在算同款）
sin_az, cos_az [2,H,W]  每像素方位角（sin/cos 编码避免 2π 跳变，且旋转连续）
（CoC 提示 r̃ 保留；ctrl_vec 的 [11][12] 两维同时保留——整块均值，向后兼容）
```

条件分工原则：**全局量走 FiLM（镜头是什么：像差系数、光圈、叶片数），
空间量走输入图（这个像素在视场哪里：H、方位角、离焦）**。第一版不上 SFT 式
逐层空间调制（多参数、易过拟合），输入通道已足够让卷积自行组合；消融期再
对比"输入通道 vs SFT 调制"。

**收益**：
- 推理整图**单次前向**，D20 的滑窗/Hann 融合/azimuth wrap 易错点全部退役；
- 训练=推理同口径（v1 滑窗推理的窗心近似误差归零）；
- 边角处网络确知"我在边角"，修复不与猫眼/旋焦形态打架。

**synth 配套改动与一个诚实的限制**：训练裁块的 GT 仍由**单 PSF 视场近似**渲染
（块内 H 恒定），第一步只把条件接口从标量换成（常数）图——网络学到的映射不变，
但接口已就位；第二步把合成 GT 升级为**块内 2–4 个 H 子带分别渲染再拼接**，
让块内 H 真实变化、条件图真实非常数。子带间用视差层同款的 tent 权重过渡，
避免硬接缝。这个两步走防止"条件图说 H 在变、GT 却是单 PSF 渲的"自相矛盾。

**验收**：合成边界 L1 不劣于 v1 滑窗版；整图前向下 swirl_catseye 预设在画幅
四角的视觉一致性（点阵图复核，同 D13 方法）；e2e demo 代码量净减。

### 4.2 P1b — PSF 描述子图注入（论文级创新点）

**改什么**：渲染器算每层（离焦档）× 每 H 桶的 PSF 时，顺手提取低维**解析描述子**：

```
r_eq      等效半径（√(二阶中心矩迹/2)）—— 模糊量
elong     log(σ_major/σ_minor)        —— 猫眼/像散拉伸度
sin2φ,cos2φ  拉伸取向（轴向量 mod π，用 2φ 编码）
ring      外环/内盘亮度比             —— 肥皂泡/奶油形态
T         relative_transmission(H)    —— 边角失光
```

注意这正是 [optics/decoupling.py](optics/decoupling.py) 解耦签名的同一套统计量
（D11）——**评测指标与网络条件共用一个物理词汇表**。

**怎么变成逐像素图**：描述子是 (层离焦, H 桶) 表格 → 每像素按其连续 CoC 值
（来自视差与 d_f/K）与 H_map 在表上双线性插值 → 6 通道描述子图。
guide 通道 11 → 17。

> ⚠️ 与 D9"双环鬼影"的区别：D9 禁止的是 **PSF 图像**在离焦轴上插值（两个不同
> 尺寸的环线性混合 ≠ 中间尺寸的环）；这里插值的是**标量统计量**，r_eq/elong/ring
> 都随离焦/像高平滑单调变化，插值是安全的。这一点写论文时也要说清。

**为什么有效（机理预期）**：v1 网络见到"边界附近一圈异常颜色"只能按平均策略修；
有了描述子，它知道这圈瑕疵下面的散景盘是"半径 30px、向 45° 拉伸 2:1、带亮边"
——修复（如何延拓盘边缘、如何处理亮边断裂）可以按核形状分化。

**成本**：PSF 本来就在算，统计是几个 reduce（毫秒级）；参数 +~5k（conv0 变宽）。

**消融设计**：{无描述子} vs {仅标量 ctrl_vec} vs {+描述子图}，在强像差预设
（soap_bubble / swirl_catseye）的边界 L1 与风格保持度上对比。

### 4.3 P1c — 黑盒嵌入对照组（写论文用，不进主线）

复刻 BokehOrNot/NAFBET 的条件方式（镜头 one-hot + 光圈数值嵌入）作为消融对照：
预期物理系数条件在**未见系数组合**（如训练没见过的 W040×vignette 搭配）上平滑
泛化、one-hot 不能——这是 RQ1"连续可控"的直接证据，也先手回应"为什么不用
现成的镜头嵌入"的审稿质疑。

## 5. P2：骨干升级（工程提效）

### 5.1 P2a — NAFNet 块替换

把 FiLMResBlock 的 plain conv×2 替换为 NAFNet 块（LayerNorm → 1×1 扩张 →
3×3 depthwise → **SimpleGate**（通道对半相乘，替代激活函数）→ 简化通道注意力 SCA
→ 1×1 收缩），FiLM 挂在 LayerNorm 之后。依据：NAFNet (ECCV22) 在去噪/去模糊的
参数效率 SOTA；**NAFBET 已在散景变换任务上验证**该骨干（NTIRE23）。
预算：同参数下换块，96ch/48ch 不动；预期边界 L1 有感知改善（文献口径 0.3–0.5dB 级）。

### 5.2 P2b — CoC 感知注意力（stretch）

Bokehlicious 把 f 值映射为注意力掩码宽度；我们做物理一致版：1/4 分辨率上加
1–2 个窗口注意力块，**窗口半径跟随本地 CoC 图**——渗色的物理传播距离就是 CoC
半径，注意力范围与之匹配是"物理先验进结构"的又一例。参数 ~0.1M，
显存在 1/4 分辨率可控。仅当 P1+P2a 后大 CoC 处边界仍是短板时上。

### 5.3 P2c — 容量阶梯（兜底）

96→128ch（≈+0.5M，总量仍 <1.3M）。**最后才动**：加容量没有故事可讲，
且增大合成域过拟合风险（v1 在 25 张背景上的"纹理找补"教训）。

**12GB 余量的花法排序**：batch 4→8（梯度方差↓）＞ crop 512→768
（块内视场变化范围↑，正好配合 P1a 子带 GT）＞ 参数量。

## 6. P3：任务结构升级

### 6.1 P3a — 边界 alpha matte 辅助头

合成管线**已知精确 GT alpha**（前景 blob 的软边），目前这个监督信号完全没用。
改动：ARNet 误差图头旁加 1 通道 matte 头，损失 +λ_a·L1(matte, alpha_gt)
（只在边界带内计权）。第一版**不改融合公式**（仍用 m），matte 作为辅助任务
正则化共享特征 + 可视化诊断；若 matte 质量高，第二版再试 matte 参与融合。
成本 +~1k 参数、零数据成本。依据：MPIB/Dr.Bokeh 都论证"边界=遮挡问题"，
显式 matte 监督让网络对边界的理解从"哪里错了"升级为"前后景各占多少"。

### 6.2 P3b — MPIB-lite 遮挡 inpainting 分支（暂缓）

边界瑕疵的物理根源是"前景背后的内容不可见"。彻底解法是网络在边界窄带内
预测被遮挡背景的虚化外观（哲学干净：**物理管一切可知的，网络只管不可知的**）。
但这接近 MPIB/Dr.Bokeh 的主战场，参数/工程量大。触发条件：P1–P3a 全做完后，
真实照片边界仍有系统性"背景内容缺失感"。在那之前**不动**。

## 9. 实施顺序与验收门槛

```
现在      50k COCO 训练（v1 收官基线）→ e2e demo 核图，确定瓶颈归因
   │
M2 末     P1a + P1b 一次改入（输入通道 8→17 + synth 子带 GT）→ 重训 50k（~6h/次）
   │      验收：① 合成边界 L1 ≤ v1 50k 的 90%；② 整图单前向边角一致性通过；
   │           ③ 真实照片"前景纹理找补"现象消失或明显减轻
   │
M3 初     P2a NAFNet 块替换（+P3a alpha 头随车）→ 重训
   │      验收：边界 L1 再降，或同指标下参数/时延降
   │
M3 中     视收益决定：P2b CoC 注意力 / P2c 容量 ←只在边界 L1 不达标时
   │
M3–M4    P1c 消融组 + 数据集实验（VABD 光圈扫描、BETD 指纹定量、RealBokeh 微调）
```

每步独立可回滚：新通道/新块都走 configs/base.yaml 开关，老配置随时能复现 v1。
