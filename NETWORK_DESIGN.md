## 1. 设计红线（任何升级不得触碰）

**信息流向只许 physics → network，不许 network → style。**

网络的职责边界：修复"从估计深度出发的分层渲染"在**深度边界**的瑕疵
（渗色/硬边/半透明圈）。焦外风格——盘的形状、亮边、猫眼、旋焦、色差——
**全部由物理渲染器产出**，网络只能"知道"它们（作为条件），不能"画"它们。

误差图融合公式不变：

```
B = m · B_neural + (1−m) · B_phys      （m 集中在深度边界，平坦区 m≈0）
```

**为什么是红线**（D49 后更新：红线**更核心**，不因 RQ3 降级而放松）：
1. **可控性证明（RQ1，主线第一理由）**：风格若部分来自网络，像差系数就不再真正控制风格，
   解耦签名矩阵也度量不清"系数→风格"——可控性立论直接塌。**这是红线现在的首要理由。**
2. **差异化定位（主线）**：与扩散/流匹配方法的本质区别就是"风格可解释"，network→style
   等于自废武功去拼别人的主场。
3. **镜头指纹（RQ3，future / P-FP 次要理由，D49）**：对系数梯度下降反求的前提是风格完全由可微
   物理承载——此理由随 RQ3 降为 future 而降为次要，但**不影响红线由 1/2 独立支撑**。

> **D49 提醒**：把 RQ3 降为 future **不等于**放松红线。红线由 RQ1 可控性独立支撑，反而更核心。

**实操检查清单**——任何网络改动前问三个问题：
- [ ] 新输入/条件是不是**物理可计算量**（来自渲染器/深度/控制向量）？
- [ ] 网络输出的活动范围是否仍被 m 限制在边界？
- [ ] 去掉细化网，物理渲染是否仍能独立产出全部风格效果？

## 2. 现状 v1（baseline 存档）

结构（实现：[refine/network.py](refine/network.py)，684,035 参数 = 2M 上限的 34%）：

```
输入 guide = [全焦图 3 ‖ 扰动视差 1 ‖ B_phys 3 ‖ CoC 提示 r̃ 1] = 8ch
ctrl_vec 13 维 → CondEncoder(2 层 MLP, 64d) → 各块 FiLM (γ,β)
ARNet：guide↓2 → PixelUnshuffle(2) → 96ch × 3 FiLM 残差块 → 粗散景残差 + 误差图 logit
IUNet：[guide 全分 ‖ 粗散景↑] → 48ch × 2 FiLM 残差块 → B_neural
融合：B = m·B_neural + (1−m)·B_phys
```

已验证（PROJECT_STATUS §6）：
- 10k 步（占位背景池）：边界 L1 较物理基线 **−27%**；
- 50k 步 COCO 正式训练进行中（9k 步时 −17%，验证集更难、仍在下降）；
- 滑窗端到端推理（D20）：收益不因滑窗损失。

**v1 的四个已知缺陷**（升级路线的靶子）：
1. (H_field, azimuth) 是**整块一个标量**走 FiLM → 训练时假装整块在画幅同一点，
   推理被迫滑窗（D20），且窗内 H 恒定近似在**画幅边角误差最大**——而那里恰是
   猫眼/旋焦最强处；
2. 网络对**局部 PSF 形状盲**：只知系数数值，不知"此处的核长什么样"，
   不同像差风格下的边界修复策略无法分化；
3. plain conv 残差块的参数效率已被 NAFNet 系明显超越（同预算性能差距实测约 0.3–0.5dB 级）；
4. 合成管线的 **GT alpha 监督信号完全未用**（白白扔掉的免费监督）。

## 3. 文献坐标与空位（2026-06 调研结论）

**条件注入机制的演进线**（散景渲染 + 相邻领域）：

```
镜头 one-hot + 光圈数值嵌入          BokehOrNot / NAFBET (NTIRE23)
  → f 值 → 注意力掩码宽度            Bokehlicious (ICCV25)
  → 文本/控制 cross-attention        BokehFlow / Generative Refocusing (2025)
  → Latent PSF 先验（VQVAE 学码本）  OmniLens++ (2025.11，像差【校正】域)
  → ??? 像差系数 + PSF 形状条件      ← 空位，本项目的位置
```

**关键空位**：没有任何散景渲染工作让网络以**实际 PSF 形状/像差系数**为条件——
最接近的 OmniLens++ 在反问题（去像差）侧，且因为"盲"设定只能用 VQVAE 学 PSF
潜表征；我们是**非盲**的（PSF 自己渲染的），可以直接解析提取，更干净。

**撞车检查**（2025.11–2026.06 新作）：BokehFlow（流匹配、焦点+强度控制）、
Generative Refocusing（DeblurNet+BokehNet、EXIF 条件）、BokehDepth——全部停留在
"虚多少/虚哪里"层面，无人做像差系数级控制。**市场佐证**：Voigtländer 2026.02 发布
带球差控制环的实体镜头、Lomography Nour Triplet 的 SA 旋钮——"球差可调"有真实
摄影需求，可写进论文 motivation。

## 4. P1：条件机制升级（创新点核心，最优先）

### 4.1 P1a — 空间物理条件图

**改什么**：guide 从 8ch 扩到 11ch，新增三个**逐像素物理图**：

```
H_map          [1,H,W]  每像素归一化像高（render_tiled 的 T(H) 插值已在算同款）
sin_az, cos_az [2,H,W]  每像素方位角（sin/cos 编码避免 2π 跳变，且旋转连续）
（CoC 提示 r̃ 保留；ctrl_vec 的 [11][12] 两维同时保留——整块均值，向后兼容）
```

条件分工原则：**全局量走 FiLM（镜头是什么：像差系数、光圈、叶片数），
空间量走输入图（这个像素在视场哪里：H、方位角、离焦）**。第一版不上 SFT 式
逐层空间调制（多参数、易过拟合），输入通道已足够让卷积自行组合；消融期再
对比"输入通道 vs SFT 调制"。

**收益**：
- 推理整图**单次前向**，D20 的滑窗/Hann 融合/azimuth wrap 易错点全部退役；
- 训练=推理同口径（v1 滑窗推理的窗心近似误差归零）；
- 边角处网络确知"我在边角"，修复不与猫眼/旋焦形态打架。

**synth 配套改动与一个诚实的限制**：训练裁块的 GT 仍由**单 PSF 视场近似**渲染
（块内 H 恒定），第一步只把条件接口从标量换成（常数）图——网络学到的映射不变，
但接口已就位；第二步把合成 GT 升级为**块内 2–4 个 H 子带分别渲染再拼接**，
让块内 H 真实变化、条件图真实非常数。子带间用视差层同款的 tent 权重过渡，
避免硬接缝。这个两步走防止"条件图说 H 在变、GT 却是单 PSF 渲的"自相矛盾。

**验收**：合成边界 L1 不劣于 v1 滑窗版；整图前向下 swirl_catseye 预设在画幅
四角的视觉一致性（点阵图复核，同 D13 方法）；e2e demo 代码量净减。

### 4.2 P1b — PSF 描述子图注入（论文级创新点）

**改什么**：渲染器算每层（离焦档）× 每 H 桶的 PSF 时，顺手提取低维**解析描述子**：

```
r_eq      等效半径（√(二阶中心矩迹/2)）—— 模糊量
elong     log(σ_major/σ_minor)        —— 猫眼/像散拉伸度
sin2φ,cos2φ  拉伸取向（轴向量 mod π，用 2φ 编码）
ring      外环/内盘亮度比             —— 肥皂泡/奶油形态
T         relative_transmission(H)    —— 边角失光
```

注意这正是 [optics/decoupling.py](optics/decoupling.py) 解耦签名的同一套统计量
（D11）——**评测指标与网络条件共用一个物理词汇表**。

**怎么变成逐像素图**：描述子是 (层离焦, H 桶) 表格 → 每像素按其连续 CoC 值
（来自视差与 d_f/K）与 H_map 在表上双线性插值 → 6 通道描述子图。
guide 通道 11 → 17。

> ⚠️ 与 D9"双环鬼影"的区别：D9 禁止的是 **PSF 图像**在离焦轴上插值（两个不同
> 尺寸的环线性混合 ≠ 中间尺寸的环）；这里插值的是**标量统计量**，r_eq/elong/ring
> 都随离焦/像高平滑单调变化，插值是安全的。这一点写论文时也要说清。

**为什么有效（机理预期）**：v1 网络见到"边界附近一圈异常颜色"只能按平均策略修；
有了描述子，它知道这圈瑕疵下面的散景盘是"半径 30px、向 45° 拉伸 2:1、带亮边"
——修复（如何延拓盘边缘、如何处理亮边断裂）可以按核形状分化。

**成本**：PSF 本来就在算，统计是几个 reduce（毫秒级）；参数 +~5k（conv0 变宽）。

**消融设计**：{无描述子} vs {仅标量 ctrl_vec} vs {+描述子图}，在强像差预设
（soap_bubble / swirl_catseye）的边界 L1 与风格保持度上对比。

### 4.3 P1c — 黑盒嵌入对照组（写论文用，不进主线）

复刻 BokehOrNot/NAFBET 的条件方式（镜头 one-hot + 光圈数值嵌入）作为消融对照：
预期物理系数条件在**未见系数组合**（如训练没见过的 W040×vignette 搭配）上平滑
泛化、one-hot 不能——这是 RQ1"连续可控"的直接证据，也先手回应"为什么不用
现成的镜头嵌入"的审稿质疑。

## 5. P2：骨干升级（工程提效）

### 5.1 P2a — NAFNet 块替换（✅ 已实现，D32）

把 FiLMResBlock 的 plain conv×2 替换为 NAFNet 块（LayerNorm → 1×1 扩张 →
3×3 depthwise → **SimpleGate**（通道对半相乘，替代激活函数）→ 简化通道注意力 SCA
→ 1×1 收缩），FiLM 挂在 LayerNorm 之后。依据：NAFNet (ECCV22) 在去噪/去模糊的
参数效率 SOTA；**NAFBET 已在散景变换任务上验证**该骨干（NTIRE23）。

**实现要点（D32）**：`refine/network.py` 加 `LayerNorm2d`/`NAFFiLMBlock`/`_make_block`
工厂，`block_type` 开关贯穿 ARNet/IUNet/RefineNet（`film_res` 复现 v1）。
**实测 NAF 块在同 ch(96/48) 下约半参**（706,917→364,005）——depthwise+SimpleGate
的效率体现。两种对照：① naf@96/48（半参，测"半参不掉点"）；② naf@144/72（712,821，
≈同预算，测"同预算更好"，§5.1 本意）。残差缩放 β/γ 初始化=1（非 NAFNet 的 0，
避免首步阻断卷积/FiLM 梯度；稳定起点由近零 head 保证）。
首个实验 train_run7 = naf@144/72 同预算，验收：合成边界 L1 ≤ film_res 基线。

### 5.2 P2b — CoC 感知注意力（stretch）

Bokehlicious 把 f 值映射为注意力掩码宽度；我们做物理一致版：1/4 分辨率上加
1–2 个窗口注意力块，**窗口半径跟随本地 CoC 图**——渗色的物理传播距离就是 CoC
半径，注意力范围与之匹配是"物理先验进结构"的又一例。参数 ~0.1M，
显存在 1/4 分辨率可控。仅当 P1+P2a 后大 CoC 处边界仍是短板时上。

### 5.3 P2c — 容量阶梯（兜底）

96→128ch（≈+0.5M，总量仍 <1.3M）。**最后才动**：加容量没有故事可讲，
且增大合成域过拟合风险（v1 在 25 张背景上的"纹理找补"教训）。

**12GB 余量的花法排序**：batch 4→8（梯度方差↓）＞ crop 512→768
（块内视场变化范围↑，正好配合 P1a 子带 GT）＞ 参数量。

## 6. P3：任务结构升级

### 6.1 P3a — 边界 alpha matte 辅助头

合成管线**已知精确 GT alpha**（前景 blob 的软边），目前这个监督信号完全没用。
改动：ARNet 误差图头旁加 1 通道 matte 头，损失 +λ_a·L1(matte, alpha_gt)
（只在边界带内计权）。第一版**不改融合公式**（仍用 m），matte 作为辅助任务
正则化共享特征 + 可视化诊断；若 matte 质量高，第二版再试 matte 参与融合。
成本 +~1k 参数、零数据成本。依据：MPIB/Dr.Bokeh 都论证"边界=遮挡问题"，
显式 matte 监督让网络对边界的理解从"哪里错了"升级为"前后景各占多少"。

### 6.2 P3b — MPIB-lite 遮挡 inpainting 分支（暂缓）

边界瑕疵的物理根源是"前景背后的内容不可见"。彻底解法是网络在边界窄带内
预测被遮挡背景的虚化外观（哲学干净：**物理管一切可知的，网络只管不可知的**）。
但这接近 MPIB/Dr.Bokeh 的主战场，参数/工程量大。触发条件：P1–P3a 全做完后，
真实照片边界仍有系统性"背景内容缺失感"。在那之前**不动**。

### 6.3 几何修正型网络（Plan B：找补的结构性根治，D24）

**触发条件**：run4（D23 背景细薄层）后真实照片仍残留找补。**这不是换路线，是换
网络的"修复方式"**——物理渲染器仍承载全部风格，红线不变。

**找补的结构性根源**：当前融合 `B = m·B_neural + (1−m)·B_phys` 里，B_neural 是
网络**直接输出的颜色图** → 网络物理上有能力把全焦纹理画回去，找补永远是一个
结构性诱惑，只能靠数据/门控"压"，压不干净。

**改法**：把网络输出从"画颜色"换成"**预测几何**"：

```
当前:   网络 → B_neural(颜色)  → m 融合         ← 网络能画像素，找补是结构性诱惑
Plan B: 网络 → 边界 alpha matte / 深度修正(纯几何量)
        → 物理渲染器拿【修正后的几何】重新渲染
        → 网络从未接触颜色通道，物理上无法找补纹理
```

网络只回答"此边界像素前景占多少、背景占多少 / 深度该修正多少"，由**可微物理
渲染器**用修正后的 alpha/深度重渲。**网络结构上没有画像素的能力 → 找补从根上
不可能发生**。这是 §1 红线"network 不许画 style"最干净的实现，哲学同 §6.2
（物理管可知、网络管不可知），且 matte GT 在合成管线免费（§6.1 已指出）。

**与 P3a/P3b 的关系**：介于二者之间——比 P3a（matte 仅辅助头、不改融合）激进，
比 P3b（MPIB 重 inpainting 分支）轻。代价：需把"修正后几何"可微地喂回渲染器
重渲（渲染器已可微，工程量中等）。

#### 6.3.1 v0 实施记录（2026-06-18，D29 敲定 + 落地）

run4 验收找补未根除 → 触发 Plan B。两个待定结构定为：**边界 alpha matte** +
**边界窄带局部重渲**（用户拍板，D29）。**v0 机制（matte 选择式重渲）**：

```
B_fg = 标准物理渲染(= B_phys)；band = boundary_band（D22/D23 同一套）
B_bg = render(image, push_band_to_background(disp))  # 带内视差压到局部背景→重虚化
B = band·(α·B_fg + (1−α)·B_bg) + (1−band)·B_phys      # α = 网络 1ch matte
```

- **找补结构上不可能**：B_fg/B_bg 均渲染器产物 → 输出恒为两张已模糊物理图的凸组合，
  永不可能比 B_phys 更锐。真背景 α→0 取 B_bg(虚)。红线由软监督升级为结构约束。
- **落地**（各 smoke 通过）：`renderer.push_band_to_background`/`foreground_occupancy_gt`、
  `RefineNet(matte_mode=True)`（1ch matte 头，**706,917 参数** < v1 同条件 719k）、
  `synth.matte_gt`（前景占比真值）、`train.refine_forward` matte 分支、`e2e_demo.refine_full_image_matte`。
- **关键踩坑：matte 直监必须用 BCE-on-logits，不能用 L1**。L1(α,α_gt) 的 logit 梯度 ∝
  σ'(z)=σ(1−σ)，在 α 饱和(→0)处消失：recon 早期把 α 压到 0 后 L1 再也拉不回来
  （train_probe_matte 实证 ᾱ→0.000 卡死 400 步不恢复）。改 BCE 后 logit 梯度=α−tgt
  不消失 → ᾱ 0.10→0.32 收敛到真值 0.34。
- **synth 边界 L1 不是 Plan B 的合适判据**：该指标在合成验证集上由【前景边缘】误差
  主导（α↑则指标↓，v0 无法修前景 bleed），而 Plan B 的目标=消除【背景找补】。
  v0 验收主判据 = **e2e 21.jpg 找补是否消失**（D29 ②）。
- **v1 优化方向**（v0 证明根治找补后）：① 2-layer 精确 over-composite（前景层 (I,α,d_fg)
  over 背景层 (I,1,d_bg)，能同时修前景 bleed，不止"选择"）；② B_bg 的腐蚀从 1/4 res
  块状改精细；③ 局部 crop 重渲省算力（v0 是整图渲 B_bg+带内 splice，结果等价）。

## 11. 网络职责再界定（2026-06-22 路线评估结论，D49 / D50）

> 本节是 D49（RQ3 降级）后对细化网角色的重新厘定，源自多轮压力测试（要不要网络 /
> 前移还是后置 / 能否用网络做深度重映射）的收敛结论。**标注为"评估结论 / 待验证"——
> 是设计指引，不立即重训。** §1 红线不变（反而更核心）。

### 11.1 一句话结论

**细化网必要，但职责收窄**：它的真正活 = **合成几何不可知的边界内容**（被遮区域
disocclusion 外观、过渡带颜色）+ **对不完美深度鲁棒**；它**不**承担：修全局深度错、
修渲染阶段遮挡、画风格。后两者外移到深度侧（冻结先验）与渲染器内部（Dr.Bokeh）。

### 11.2 为什么"必要"——但 5% 这个数字会骗人

沿用 BokehMe 立论：细化网是为"对 imperfect disparity 鲁棒"而存在的。但 train_run6 的
合成边界 L1 净收益 −0.0033(~5%) 是在**深度近乎完美的合成验证集**上测的（输入侧只加了
局部扰动），**恰恰是网络最不重要的 regime**，严重低估其在真实烂深度上的价值。
→ **细化网的判据必须换到真实图失败模式，不能只看合成边界 L1。**

### 11.3 深度误差分两种，决定网络能做什么

| 深度误差 | 例子 | 细化网能修吗 | 归属 |
|---|---|---|---|
| **局部边界错** | 边缘软坡 / 偏几像素 | ✅ 能（RGB 引导对齐边界，非重估深度）| 细化网 |
| **全局结构错** | 脸-树 merge、错序、背景判近 | ❌ 不能（修它 = 造一个比 DA V2 强的单目模型，0.7M 头做不到）| 深度侧先验 / 已知限制 |

### 11.4 位置：后置 + 输出几何（不要前移成深度修正头）

- **保持在渲染器之后**：网络需看到 B_phys 才知道"哪里错了、要合成什么"；前移的深度网是"瞎"的。
- **输出几何（matte / 占用），不输出颜色**：守红线，找补结构上不可能。
- **不做"前移成可训练深度修正头"**：那既进轻量预算、又撞"打败 DA V2"墙（病态：单图恢复度量深度）。
  "前移"仅可作**可选的 RGB 引导边缘锐化**修局部软边，**替代不了后置网**。

### 11.5 不要 overload：三件事分给三个地方

| 病 | 根因 | 最该在哪治 |
|---|---|---|
| 全局深度错 | 输入侧（DA V2）| **深度侧冻结先验**：DA V2-Large（已用）；人像可选**冻结 matting / 分割先验**（同 DA V2 口径、不算可训练参数、不破轻量，D25 先例）；否则记为已知限制 |
| 渲染阶段遮挡（色渗 / tent 漏光，D34）| 渲染算子 | **渲染器内部** Dr.Bokeh 式层内 O_l + 层间 V_l alpha；非网络的活 |
| 边界不可知内容（disocclusion 外观）| 物理不可知 | **细化网**（唯一只能学的部分）|

### 11.6 高 ROI 待办：修训练 / 真实失败模式失配

`data/synth.py` 当前只对视差加**局部扰动**（边界形变 / 高斯模糊 / 低频噪声），而真实
DA V2 的致命错是**全局**（merge / 错序）→ **网络从没在训练里见过它在真实图上要修的东西**，
这很可能就是它"必要却在真实图不够给力、换 NAFNet 也只打平"的根因。
→ **给合成扰动加全局 / 结构性扰动**（层间 merge、深度压缩、错序），比换骨干、加容量 ROI 高得多。

### 11.7 前景 / 背景虚化尺度（顺带结论）

不追深度重映射网络（D39 已撤回 γ，全局 γ 会推 d_f 虚掉主体）。前景天生比背景虚得多
是**物理正确**（CoC ∝ |1/z − 1/z_f|，背景虚化有上界、前景无）。背景偏弱的真问题是
DA V2 无度量尺度（远景压缩）：靠 **d_f / K + 上限**控制（相对模糊，BokehMe 传统）；
想要物理正确比例可换**冻结 metric depth 先验**；前 / 背景虚化量可作**明确标注非物理**的审美旋钮。

### 11.8 落地顺序（待验证，不急于重训）

1. **with / without-net 消融**（在真实图失败模式上，不只看合成边界 L1）—— 量化网络真实价值。
2. **合成扰动升级**（§11.6）—— 让网络见到真实全局错，重训。
3. 视收益再决定：**渲染器 Dr.Bokeh 遮挡**（§11.5）/ **人像 matting 冻结先验**（全局错兜底）。
4. 一切以红线（§1）为前提：网络永远只输出几何、永远不画风格。
