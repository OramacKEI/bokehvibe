## 1. 项目使命（务必牢记）

从**单张全焦图像**生成**浅景深（散景）图像**。区别于所有现有方法的核心创新：把镜头的**光学像差**（球差、彗差、场曲、像散、口径蚀）建模为一组**连续、物理可解释、可解耦**的控制旋钮，并保持**单张消费级 GPU 可训练**的轻量体量。

一句话定位：现有方法能调"虚多少、虚哪里"，本项目要让用户能调"**虚成什么风格（像差风格）**"。

**主线研究问题（当前主攻，见 DECISIONS D49）：**
- **RQ1 可控性**：低维像差系数能否对焦外风格做到连续且解耦的控制？
- **RQ2 轻量性**：能否在单张消费级 GPU 上训练，且保真度不输 SOTA？

**future 阶段研究问题（主线投稿后的新阶段 P-FP，D49 已降级，不作硬约束）：**
- **RQ3 真实性锚定**：能否从真实镜头样张反求像差系数（lens fingerprint），复刻其焦外风格？
  —— **降为 future work**。理由（D49）：sim-to-real 风险最高、最未验证（单波前对真实多片镜组的保真度
  仅在自合成目标上验证过 = 循环），且差异化叙事不依赖它。代码 `render/fingerprint_demo.py` 保留待重启。

差异化策略：**不与扩散方法拼整图真实感 SOTA**，以"可控性 + 物理可解释 + 轻量"三点取胜；
"像哪支镜头（指纹反演）"是 future 阶段（P-FP）的锦上添花应用，**不是立论根基**。

## 2. 系统架构（完整管线）

```
全焦图像 I
   │
   ▼
[冻结] Depth Anything V2-Base (97M, 仅推理)  ──►  相对视差 D（近大远小）
   │
   ▼
带符号 CoC 图 + 占用感知分层      r_i = K·|D_i − d_f|，符号由 (D_i − d_f) 决定
   │
   ▼
[核心] 像差化可微渲染器  ◄── 控制向量 c = {焦平面 d_f, 光圈 K, 像差系数 a}
   │        （pupil → FFT → PSF → 空变散射/合成）
   ▼
边界细化网络（唯一可训练部分, ~1–2M）  ◄── c（FiLM 条件注入）
   │
   ▼
误差图融合  ──►  浅景深输出 B
```

**模块权重属性（关键，影响一切实现决策）：**

| 模块 | 权重 | 说明 |
|---|---|---|
| Depth Anything V2-Base | **冻结，仅推理** | 97M，绝不训练它（D25：默认 Base；不算可训练参数，不碰轻量约束。可切回 Small/vits） |
| 像差化可微渲染器 | **无权重** | 纯可微物理层 |
| 边界细化网络 | **唯一可训练，~1–2M** | 参考 BokehMe++ 的 ARNet + IUNet；职责=合成边界**不可知内容** + 抗深度误差，**不画风格（红线）、不修全局深度**（D50 / NETWORK_DESIGN §11）|

控制向量 `c` **同时**进入渲染器（物理塑形 PSF）和细化网（FiLM 条件注入），保证两者一致。

## 3. 技术内核：像差化 pupil→PSF 生成器

这是项目的技术心脏，也是最主要的工程量所在。归一化光瞳坐标 `(ρ, θ)`，`ρ ∈ [0,1]`；归一化像高 `H`（到画面中心距离）。

**波前像差（相位项）：**
```
W(ρ,θ; H) = W020·ρ²            (离焦 defocus)
          + W040·ρ⁴            (初级球差 spherical)
          + W060·ρ⁶            (高阶球差 spherical-2：盘内带状/真 nisen 二线, D34)
          + W131·H·ρ³·cosθ     (彗差 coma)
          + W222·H²·ρ²·cos²θ   (像散 astigmatism)
          + W220·H²·ρ²          (场曲 field curvature)
```

**光瞳振幅（遮罩项）：**
```
A(ρ,θ; H) = M_aperture(ρ,θ)          (n 边形/圆形通光 + 叶片曲率 blade_curvature[0直边→1圆,角仍尖,D44] → 多边形散景)
          · M_vignette(ρ,θ; H)       (随像高平移的猫眼截断, 沿用 BokehMe++ K0/K, z_l 推导)
          · M_apod(ρ) = exp(−a·ρ²)   (变迹 STF/APD：径向透过率衰减 → 盘边柔化无硬边, D33)
```

**复光瞳与 PSF（两种模式，`RenderControl.psf_mode`，见 D40）：**
```
波动光学 'wave'：P = A·exp(i·2π·W)；  PSF = |FFT2D{P}|²       （软边；星芒/衍射；小离焦近焦更准）
几何光学 'geom'(默认)：落点(px) = 2.02·∇W；按 A² 散射 splat    （硬边散景盘；像差焦散；更快；无采样上限）
```
**默认 `geom`**：散景盘外缘是【硬边】（落点边界=光瞳边界），soap 亮边环/cream 亮心/猫眼/多边形/coma/astig
均由 ∇W 与孔径 A 自然涌现。波动光学 PSF 的盘外缘恒为盘半径的 ~13% 软边（受采样上限只能仿真小离焦、停在衍射
regime），故视觉散景盘用 geom；星芒/衍射或指纹精细反演(梯度更强)用 wave。两模式标定一致（盘半径=3.93·W020）。

**色差（逐通道）：** R/G/B 各算一次 PSF。LoCA = 各通道 W020 微小偏移（焦前偏紫/焦后偏绿自动涌现）；LaCA = 各通道 PSF 随像高的**径向平移**（∝H，沿 +x 字典约定；物理上 LaCA 是各通道放大率不同导致的 PSF 中心错位，**不是缩放**——缩放更接近球色差，见 DECISIONS D10）；**球色差 spherochromatism = 各通道 W040 微小偏移**（球差随波长变 → 盘边亮环逐通道不同 → 紫/绿镶边随离焦变化；是 LoCA"只移盘心"的形态版补充，D33）。逐通道成本仅 3× PSF，默认启用。

**可微性与效率（重要实现策略）：** `|FFT{P}|²` 对所有系数可微，同时支撑端到端训练与镜头指纹标定（对 `a` 梯度下降匹配目标镜头）。PSF 仅随 `(像高分桶, 带符号离焦, 像差向量 a)` 变化。**训练/标定时按层现场算 PSF**（渲染器按视差分层后每层离焦量确定，直接对确定值算，每前向约几百个 256² FFT，GPU 毫秒级）；**PSF 字典/查找表仅作推理缓存**（镜头系数固定时预计算加速）。不要在训练路径上做字典的离焦轴插值——两个不同尺寸的环状 PSF 线性混合会产生"双环鬼影"而非中间尺寸的环（DECISIONS D9）。

**离焦标定与采样上限（接渲染器的桥梁）：** 像素 CoC 半径与波前离焦的换算已标定：`r_px ≈ 4·W020 (waves)`（实测斜率 3.93，见 `outputs/psf_test/defocus_calibration.json`，由 `optics/calibrate.py` 产出）。相位采样上限 **W020 < N/8 waves**（光瞳网格 N=256 → 约 32 waves → 最大盘半径约 128px），超限会混叠 wrap-around，`pupil_to_psf` 内置 guard 告警。

**光谱平均（盘面均匀性，见 DECISIONS D16）：** 单色相干仿真的离焦 PSF 盘内有高对比菲涅耳干涉环，但真实相机是宽光谱+部分相干+传感器积分，散景盘接近均匀。`rgb_psf` 默认对每通道在 ±6% 带宽内取 3 个子波长（相位 ∝ 1/λ）加权平均：环纹互相抵消、盘面均匀，而盘的尺寸/形状/边缘特征保留。成本 3×FFT。

**口径蚀的能量分量：** PSF 逐个归一化为能量分配核（sum=1），猫眼的"边角失光"分量由 `pupil.relative_transmission` 的 T(H) 单独承载，渲染合成时乘回（DECISIONS D10）。

**渲染器设计约定（M2，见 DECISIONS D8）：** 采用**分层 gather + tile 近似**：按视差分 K 层；图像切 tile（如 64px），每 tile 用其中心 (H, 方位角) 的 PSF（从 +x 约定旋转一次到 tile 朝向）做 FFT 卷积；层间前后 alpha 合成 + 归一化处理遮挡。HDR 高光恢复在卷积前做（支撑 glow/星芒，参考 BokehMe 的 highlight 处理）。Dr.Bokeh 式精确遮挡是改进项而非起点。

**高光权重重分配（WR，见 DECISIONS D27/D28，焦外风格化要点）：** 8-bit/有限动态范围原图里的过曝高光已能量溢出+带柔光晕（看着比真实大、边缘渐变），直接卷积会把柔边叠加放大成"高斯波"弥散圆——扁平糊团、无层次（BokehMe++ Fig.3 的 w/o WR）。**散射前**必须在线性域把高光能量分布掰成"方波"：锐利掩膜定 footprint（~1px 陡边）→ 内部压成均匀平顶（各像素自身色度 × 统一峰值，保色相）→ 内外掩膜混合。`PSF ⊛ 平顶方波 ≈ PSF 本身`，弥散圆才有锐利边缘+均匀内部+红蓝叠加分层折射感；这也是 PSF 形状特征（球差亮环/猫眼/多边形）能在最终图上被看清的前提（软边高光会把它们抹糊）。落地于 `render/renderer.py:redistribute_highlights`。**高光检测用【通道最大值】而非亮度均值**（过曝是逐通道饱和的——饱和红光亮度均值低会漏检，导致彩色光斑偏暗，D28）。**能量放大默认关、散景球亮度靠 gamma 域亮斑化（D42/D44/D45 重要修正，覆盖早期 D38）：** D38 曾让平顶按饱和度恢复 HDR（`peak=1+gain·sat`，gain 数百）以提亮散景球——但**从 LDR 猜测放大几十倍**会把非点光源亮区（天空/雪/脸，绝对值超阈值）误判放大成**假弥散圆**、真点光源放大到**溢出死白**（用户反馈）。故 **demo/e2e 默认 `highlight_gain=0`**（WR 仍做方波【形状】重塑，但不做能量放大）。散景球的"亮"改靠 **BokehMe 式 gamma 域亮斑化**：渲染在 `image**γ` 准线性域（标准 γ=2.2，见 `render(image**γ)**(1/γ)`），暗背景在 `**γ` 域趋近 0 → 散景盘归一化后**保持亮点本身亮度、不被背景稀释、绝不超过原值（不溢出）、无模糊区恒等**。`highlight_gain>0` 保留为**可选温和增强**（仅已知强点光源的纯夜景；用独立 `highlight_hdr_thresh≈0.92` 只对真饱和 d→1 放大，避免误判中等亮区）。**LDR 输入无点光源真实 HDR 信息，散景球偏暗是固有限制——不用非物理高 γ 压对比来伪造亮度**（高 γ 的 `**(1/γ)` 会把球差盘内对比从线性 6.5× 压平到显示 1.6×，抹掉 soap 亮环/cream 亮心，D45）。

**HDR 软肩色调映射（重叠盘溢出，见 DECISIONS D41）：** 散景盘是【加性 HDR 光】——同层 `out=conv(lin,psf)`，多个盘重叠处线性能量**相加**（物理正确：两路灯叠加确实更亮、可饱和到白）可远超 1.0（实测密集灯串 40% 像素 >1）。末端不能硬裁（`clamp` 上界）：超 1 部分被砍成同一片**平白块**、丢盘边界与红蓝叠加层次。`linear_to_srgb` 在 gamma【之前】插入软肩 `_soft_shoulder(L,knee=0.8)`：`L≤knee` 恒等（不动主体/正常曝光）、`L>knee` 指数滚降渐近逼近 1（C1 可微、永不超过）。**不用全局 Reinhard**（`L/(1+L)` 会把中间调一起压暗、改变整张曝光）——软肩是"保 LDR、只救 HDR 溢出"的最小干预，符合标准成像管线（HDR 加性合成 → tone map → 显示域 → OETF）。`highlight_tonemap` 默认开、仅 `gain>0` 启用（无高光时内容本就 ≤1、跳过保持 byte 级不变）。

**光瞳 apodization 抑制相干衍射边环（见 DECISIONS D28，真实性硬点）：** `|FFT{硬边光瞳}|²` 是相干仿真，会在散景盘几何边界处产生一圈伪【衍射亮边环】（理想无像差镜头也有，约内部 1.25×）——真实相机宽光谱+部分相干+点源有限角直径会抹掉它，理想盘边缘应中性（亮环只应由球差 W040 产生）。对策：光瞳边缘做轻度 apodization（`pupil.DEFAULT_SOFTNESS=0.04`，等效部分相干平滑），把伪环压到 ~1.10× 而不动像差特征。分工：**光谱平均(`psf.SPECTRAL_SAMPLES`)抹【盘内】菲涅耳环，apodization 抹【盘边】相干环**。注意：理想盘的"亮边"若未抑制易被误判为"球差没解耦"，但 PSF 层面 cream→亮心/soap→亮环/ideal→平本就分离（decoupling 矩阵对角占优）。

**PSF 裁剪/halo 须按完整像差外缘（见 DECISIONS D26）：** 现场算 PSF 的自适应裁剪窗口与 tile halo，半径必须计入【所有】像差项（`4|W020|+8|W040|+6H|W131|+4H²(|W222|+|W220|)`，由 `render/renderer.py:psf_extent_px` 统一估），不能只按离焦 `4|W020|`——否则近焦面强球差的肥皂泡亮边会被裁掉、重归一化又把截断悄悄掩盖。

**效果如何由参数涌现（实现时对照验证）：**

| 焦外风格 | 主控参数 | 档位 |
|---|---|---|
| 奶油柔散 | 轻球差欠矫正 W040<0（点缀亮心）+ **变迹 apodization**（柔边填充=真虚化） | 第一档（D48）|
| 肥皂泡亮边环 | 球差过矫正 W040 > 0 | 第一档 |
| 二线 nisen（盘内带状双线） | 高阶球差 W060（与 W040 反号） | 第一档扩展（D34）|
| 前景/背景形态反转 | 球差符号 | 第一档（标志特征） |
| 双高斯单侧亮边 | 球差亮环 × 视场相关猫眼截断 | 第一档（两旋钮乘积免费涌现） |
| 旋焦 swirl | 视场相关猫眼（+像散） | 第一档 |
| 多边形散景 | M_aperture 叶片数 n_blades + **叶片曲率 blade_curvature**（0=直边→1=圆，角仍尖，D44） | 第一档 |
| 锐利亮斑散景 / glow | 高光权重重分配 WR（平顶方波）+ gamma 域亮斑化（默认）；纯夜景可选温和 highlight_gain | 第一档 |
| 纵向色差 LoCA | 分通道 W020 偏移 | 第一档 |
| 横向色差 LaCA | 分通道视场相关径向平移 | 第一档扩展 |
| 变迹奶油焦外 (STF) | 径向透过率衰减 exp(−a·ρ²) | 第一档扩展（D33）|

> **球差符号配对（易错，见 DECISIONS D13）：** 亮边环出现在 **sign(W040) ≠ sign(W020)**
> 的一侧（边缘光线折返成焦散环），同号侧为奶油。本项目约定背景 (D<d_f) → W020<0，
> 上表的 奶油/肥皂泡 命名按**背景侧**定义；前景侧自动反转（即"形态反转"特征本身）。
> 任何对这两个效果的验证/测量都必须先确认离焦符号。

## 4. 范围边界（严格遵守，防止 scope creep）

| 效果 | 是否纳入 |
|---|---|
| LoCA / LaCA 色差、高光柔晕 | ✅ 第一档 |
| 星芒 | ⚪ 可选彩蛋 |
| **鬼影 / 完整眩光** | ❌ **范围外** |

**鬼影/眩光为何排除：** 它们是光沿非成像路径（镜组元素间多次反射、传感器杂散）传播的结果，需完整多片镜组处方做光线追踪，与本项目单光瞳→PSF 模型物理不兼容。属独立研究领域，列为 future work。若任务出现"加鬼影/眩光"的需求，应提示这超出范围。

**优先级铁律：** 第一档效果优先全做；第二档作 stretch goal，做不完则作为"框架可扩展性"展示，**不得为追第二档拖垮第一档的完成度**。

## 7. 数据策略

- **训练数据（在线合成）：** 透明前景（生成或抠图素材）+ 全焦背景，按**随机采样的像差向量 a** 现场渲染 (全焦图, 散景图) 配对。系数已知 → 提供完美可控性监督，且独立采样天然保证解耦。
- **GT 与输入要区分开（防 sim-to-real 边界崩坏）：** GT 用**已知 alpha 的精确分层合成**生成；网络输入侧只有单图 + **加扰动的深度**（模拟 DA V2 的估计误差，BokehMe 的关键 trick）——不扰动则细化网在真实深度误差下的边界必崩。
- **HDR 高光：** 合成素材需包含高亮点源/高光恢复（支撑 glow 与亮斑散景，参考 BokehMe highlight 模式）。
- **真实验证集（不用于可控性训练）：** RealBokeh（Bokehlicious 提供）、EBB! Val294、自采手机人像。
- **镜头指纹素材（future / P-FP，D49）：** 双高斯标头、Helios 44-2、Trioplan 等特色镜头失焦样张，用于 P-FP 阶段标定与定性复刻；**主线不需要**。

## 8. 损失函数

- **重建：** L1 + 感知损失（VGG / LPIPS），对合成 GT。
- **真实感（可选）：** 真实集轻量微调或对抗损失。
- **解耦：** 独立采样系数 + 配对 GT 隐式实现；必要时加解耦正则。

## 9. 评测协议（写实验代码时对照）

> **循环性措辞（论文必须主动说破）：** 可控性指标(1)(2)的 GT 由本项目自己的物理渲染器合成，
> 度量的是"输出对物理模型的**忠实度**"，不是真实感——这是可控性唯一可行的测法，
> 但若不说破会被审稿人视为循环论证。**真实感单独由(3)承担**（RealBokeh / user study）。
>
> **主线评测范围（D49）：** 主攻 RQ1/RQ2 → (1)(2)(4)(6) 为主线硬数据、(3) 真实感**降权报告**；
> (5) 镜头指纹与 (1)(2) 中"对真实镜头"的部分降为 **future / P-FP 阶段**。

1. **可控性保真度：** 给定 `(d_f, K, a)` 对合成 GT 算 PSNR / SSIM / LPIPS。
2. **解耦度（cross-talk）：** 效应签名矩阵——每个旋钮配一个专属可测签名
   （球差→外环/内盘亮度比、彗差→x 向偏度、像散→log(σx/σy)、猫眼→相对透过率 T(H)），
   逐旋钮扰动测全签名的带符号变化、按列归一化，**对角占优=解耦好**。
   PSF 层面已落地（`optics/decoupling.py`，DECISIONS D11）；M3 在图像层面复用同一套签名。
3. **真实感：** RealBokeh / EBB! Val294 指标 + 小规模 user study。
4. **效率：** 可训练参数量、FLOPs、时延、显存——对比 BokehMe++ / Bokehlicious / BokehDiff。
5. **镜头指纹（future / P-FP 阶段，D49 降级，非主线）：** 对双高斯单侧亮边、Helios 旋焦、
   Trioplan 肥皂泡的定性复刻；BETD lens-to-lens 定量。**主线评测不含本项。**
6. **消融：** 去像差控制 / 去视场相关 / **去细化网（with/without-net，D50）** / 去真实微调。

## 12. 与 Claude Code 的协作规范

- **概念坑常驻提醒：** CoC 公式 `r = K·|D − d_f|` 中的 `D` 必须是**视差（近大远小）**，不是物理深度。DA V2 输出方向正确，但归一化范围因加载方式而异——任何用到深度的地方都要先确认方向。**CoC 对视差(1/物距)线性、对物距非线性**——我们工作在视差域，故公式已是正确物理，无需为物理补非线性。背景虚化不足通常是**深度质量/对焦带过宽**问题：务必用默认 **DA V2 vitl@770**（低质量缓存会把主体与背景 merge，误判为"压缩"）；对焦容差 tol 要够紧（D39）。`remap_disparity(D,γ<1)` 幂律去压缩仅作**可选工具默认关闭**（γ<1 会推高 d_f 虚掉主体，慎用）。
- **可微性是保留的设计属性（D49 降级，非硬约束）：** 当前训练在 `no_grad` 下跑物理渲染（见 PROJECT_STATUS §3），可微性此刻只服务已降级为 future 的指纹反演；但保留它**成本极低、且为 P-FP 留门**，故渲染器/PSF 生成器仍**尽量**保持对 `a` 可微。引入不可微操作**不再一票否决**，但需标记、说明、且默认保守（勿无谓破坏可微链路）。
- **轻量是底线：** 任何会显著增加显存/参数的设计，先评估是否突破 12GB / ~2M 约束。
- **改动前先说明：** 涉及架构或核心算法的改动，先解释思路与权衡，再动手。
- **产出可验证：** 每个模块尽量配最小可视化/单测，让"对不对"能被肉眼或指标确认；关键视觉产物（深度图、PSF、散景结果）由人工过一道再推进。
- **遵守范围边界：** 见第 4 节，不主动扩张到鬼影/眩光等范围外内容。
