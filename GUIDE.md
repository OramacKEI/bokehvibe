## 1. 一句话：这个项目做什么

**输入一张普通的全清晰照片，输出一张背景虚化的"大光圈"照片——并且让你能调"虚化成什么风格、像哪支镜头"。**

手机人像模式、各种散景渲染论文都能做到"虚多少（光圈）、虚哪里（对焦）"。
但真实镜头的迷人之处在于**焦外的"性格"**：

- 老蔡司双高斯镜头的焦外有单侧亮边；
- Helios 44-2 的焦外会"旋转"（旋焦）；
- Trioplan 的焦外亮斑是"肥皂泡"（亮边圆环）；
- 现代定焦的焦外是"奶油"般化开的柔散。

这些"性格"在物理上全部来自**镜头的光学像差**。本项目把像差做成几个**连续可调的旋钮**，
让一张照片可以渲染成"任何一支镜头拍出来的样子"——甚至可以反过来，
**从某支镜头的样张里把它的像差"指纹"反推出来**，然后复刻它（**指纹反演 = 远期愿景，已降级为
future / P-FP 阶段，见 D49；当前主线聚焦"像差可控渲染"本身**）。

> 顺带一提：Voigtländer 2026 年 2 月刚发布了带**球差控制环**的实体镜头（约 ¥9 万日元），
> Lomography Nour Triplet 也有 SA 旋钮——市场在用硬件卖"球差可调"，
> 我们用软件把它做成连续、可组合、可反演的。这是论文 motivation 的现成佐证。

三个研究问题（RQ）：

> **路线更新（2026-06-22，DECISIONS D49）：主线只攻 RQ1 + RQ2；RQ3 镜头指纹反演降级为
> future work（主线投稿后的新阶段 P-FP）。** 下文"像哪支镜头 / 指纹反演"讲的是**远期愿景**，
> 当前主线 = 各类像差可控的轻量化单图散景渲染，**不把指纹当硬约束**。

| | 问题 | 检验方式 |
|---|---|---|
| **RQ1 可控性（主线）** | 几个低维系数能否**连续、解耦**地控制焦外风格？ | 解耦签名矩阵、可控性保真度 |
| **RQ2 轻量性（主线）** | 能否在**一张 12GB 消费级显卡**上训练完？ | 可训练参数 ~1–2M、显存/时延对比 |
| RQ3 真实锚定（**future / P-FP**）| 能否从真实样张**反求像差系数**（镜头指纹）？ | 自标定实验 + 真实镜头复刻 + BETD 定量（**降级 future，D49**）|

## 2. 背景知识速成

> 看懂本项目需要的全部光学 + 一点点神经网络。每小节末尾标注对应代码。

### 2.1 散景的几何：CoC（弥散圆）

对焦在距离 d_f 的物体上时，其它深度的点不会成像为点，而是摊开成一个圆斑，
叫**弥散圆（Circle of Confusion, CoC）**。离对焦面越远、光圈越大，圆斑越大：

```
r = K · (D − d_f)        r：圆斑半径(像素)   K：光圈增益   D：该点的视差
```

**易错点**：这里的 D 是**视差**（disparity，近大远小），不是物理深度。
深度网（Depth Anything V2）输出的恰好就是视差，方向正好可以直接用。
r 的**符号**区分焦前（前景）/焦后（背景）——前后两侧的虚化形态不一样，
这正是像差带来的标志性特征。

实际渲染还有一个**对焦容差 δ0**：DA V2 的相对视差会把一个主体自身拉出 0.1~0.3
的视差跨度，不加容差的话光圈一大主体内部就被虚掉。公式变为
`r = K·sign(δ)·max(|δ|−δ0, 0)`，把整个主体圈进景深带（DECISIONS D14）。

→ 代码：[render/renderer.py](render/renderer.py) `signed_coc`，标定换算 `coc_to_w020`。

### 2.2 散景的形状：PSF（点扩散函数）

CoC 只说了圆斑**多大**，没说圆斑**长什么样**。描述"一个点光源被镜头摊成什么图案"的
函数叫 **PSF（Point Spread Function）**。夜景里的散景小圆斑，就是 PSF 的"直接照片"。

理想镜头的 PSF = 均匀亮圆盘。真实镜头的 PSF 千姿百态——亮边环、猫眼形、彗星形……
**散景的"性格" = PSF 的形状**。所以本项目的核心问题就是：**怎么参数化地生成"有性格"的 PSF？**

### 2.3 生成 PSF 的物理引擎：光瞳 → FFT → PSF

傅里叶光学给了一个非常优雅的答案（这是全项目的技术心脏，代码在 `optics/`）：

```
复光瞳  P(ρ,θ) = A(ρ,θ) · exp( i·2π·W(ρ,θ) )
PSF   = | FFT2D{ P } |²
```

只需要描述镜头出瞳处的光场（振幅 A + 相位 W），做一次 2D 傅里叶变换取模平方，
就得到 PSF。两个组成部分各管一摊：

**相位 W（波前像差，单位=波长数）**——管"光的弯曲方式"：

| 项 | 名字 | 调它会发生什么 |
|---|---|---|
| W020·ρ² | 离焦 | 圆斑变大（就是 CoC，由渲染器按层自动注入） |
| W040·ρ⁴ | **球差** | **主角旋钮**：W040>0 边缘出亮环（肥皂泡）；W040<0 中心聚光（亮心）。注：真正的"奶油【柔虚化】"靠【变迹 apodization】(柔边填充盘)而非欠矫正球差(只给亮心+保留焦外结构=柔化≠虚化)，故 cream 预设=轻球差+变迹，见 DECISIONS D48 |
| W131·H·ρ³cosθ | 彗差 | 圆斑拖出彗星尾（画面边缘处） |
| W222·H²·ρ²cos²θ | 像散 | 圆斑拉成椭圆（强了就是"爆炸焦外"） |
| W220·H²·ρ² | 场曲 | 画面边缘整体跑焦 |

（H = 像高，即该点离画面中心多远，0=中心、1=对角。带 H 的项只在画面边缘发力
——这叫**视场相关**。）

**振幅 A（光瞳遮罩）**——管"光从哪里通过"：

| 项 | 调它会发生什么 |
|---|---|
| 光圈叶片数 n_blades | 圆斑变成六边形/八边形（收光圈后的多边形散景） |
| 猫眼截断 (strength, R_v) | 画面边缘的圆斑被切成柠檬形（猫眼），排成一圈 → **旋焦** |

**两个关键的"配对定理"**（踩过坑，务必记住）：

1. **亮边环出现在 sign(W040) ≠ sign(W020) 的一侧**。本项目约定背景为 W020<0，
   所以 W040>0 → 背景肥皂泡 + 前景奶油；W040<0 正好相反。"前后景形态反转"
   是球差镜头的身份证，我们的模型里它**免费涌现**（DECISIONS D13）。
2. **猫眼 × 球差亮环 相乘 = 双高斯单侧亮边**。两个旋钮的组合自动涌现第三种风格，
   这正是"物理模型"优于"风格标签"的地方。

另外两个让仿真贴近实拍的细节：**光谱平均**（单色仿真的圆斑内有干涉环纹，
按 ±6% 带宽取 3 个子波长平均后盘面就均匀了，D16）；
**色差**（R/G/B 各算一次 PSF：LoCA=各通道离焦微移 → 前紫后绿；LaCA=各通道 PSF
沿径向错位 → 边缘紫/绿镶边，注意是**平移不是缩放**，D10）。

→ 代码：[optics/pupil.py](optics/pupil.py)（W 与 A）、[optics/psf.py](optics/psf.py)（FFT+色差+guard）、
[optics/aberrations.py](optics/aberrations.py)（系数与预设）。跑 `python -m optics.visualize` 一目了然。

### 2.4 为什么"可微"这么重要

整条 `系数 → 光瞳 → FFT → PSF → 卷积 → 散景图` 的链路全部用 PyTorch 张量运算实现，
**对所有系数可导**。这买到两样东西：

- **训练**：梯度能从图像损失一路传回控制向量，细化网可以端到端学习；
- **镜头指纹（RQ3）**：把"目标镜头的样张"当 GT，对像差系数做梯度下降——
  就能**反求**出那支镜头的参数。已验证：3 个参数从故意给错的初值出发，
  420 步、11 秒收敛到误差 <1.1%（`render/fingerprint_demo.py`，D12。
  坑：必须两阶段——先低阶 (d_f,K) 再放开 W040，单阶段联合优化会掉局部极小）。

### 2.5 神经侧速成：边界细化网为什么存在、长什么样

物理渲染器拿到的视差是**估计值**（DA V2 难免有误差：物体边界不贴、过渡带发糊）。
从有误差的深度出发做分层渲染，瑕疵集中在**深度边界**：前景颜色渗进背景虚化区
（渗色）、本应柔和的轮廓出现硬边、边缘出现半透明圈。**平坦区域物理渲染是对的，
只有边界需要修**——这就是细化网的全部职责，也是它只需要 ~0.7M 参数的原因
（v1 684k → v2 719k）。

结构（BokehMe 的 ARNet+IUNet 轻量版 + FiLM 条件，[refine/network.py](refine/network.py)）：

```
v1 输入 8 通道：全焦图(3) ‖ 扰动视差(1) ‖ 物理渲染 B_phys(3) ‖ 离焦提示 r̃(1)
v2 再 +9 通道（P1）：逐像素 [像高 H, sin/cos 方位角] ‖ PSF 描述子图
                     (r_eq/拉伸/取向/亮边环/边角失光) → guide 共 17 通道
   │
   ├─ ARNet（在 1/4 分辨率工作，省算力）→ 粗神经散景 + 误差图 m
   ├─ IUNet（全分辨率，用原图细节引导）→ 精化的神经散景 B_neural
   │
   └─ 融合：B = m·B_neural + (1−m)·B_phys     （m 还乘了"边界带门控"，见下）
```

四个教学点：

- **误差图融合**是安全阀：m 只在"物理渲染出错的地方"（深度边界）升高，
  平坦区 m≈0 → 物理层的像差风格（肥皂泡/猫眼/色差…）**不会被网络洗掉**。
- **FiLM 条件注入**：13 维控制向量 c 经小 MLP 编码后，对每个残差块的特征做
  逐通道仿射 `h ← h·(1+γ)+β`。直觉：网络由此"知道"现在是什么镜头、多大光圈，
  可以对不同风格采取不同修复策略。c **同时**喂渲染器和细化网——物理层和
  神经层听同一套指挥（CLAUDE.md 第 2 节的一致性要求）。
- **v2 升级（P1）**：把"镜头是什么"（全局量：像差系数/光圈）留给 FiLM，把"这个
  像素在视场哪里、这里的散景核长什么样"（空间量：像高/方位角/PSF 形状）做成
  **逐像素物理条件图**喂进输入——网络从此能整图单次前向（不再滑窗），边角的
  猫眼/旋焦修复不再与风格打架。详见 [NETWORK_DESIGN.md](NETWORK_DESIGN.md) §4。
- **设计红线 + 血泪坑（找补）→ 已用 Plan B 根治**：信息流向只许 physics→network，网络不许
  "画"风格只许"知道"风格。**现实坑**：v1 的"误差图融合" B=m·B_neural+(1−m)·B_phys 里 B_neural 是
  网络直接输出的颜色 → 它有能力把真实照片里被正确虚化的背景树枝"抄"回锐利（"找补"）。数据修正+
  门控只能"压"不能"根治"。**现已换成 Plan B matte 选择式重渲**（D29）：网络只输出 1ch 几何 matte α，
  最终图 = 两张物理渲染（B_fg 锐 / B_bg 虚）的逐像素凸组合 → **网络结构上画不出锐利纹理，找补不可能
  发生**。train_run6 验收：真实图人脸清晰、无找补。红线/找补/Plan B 详见
  [NETWORK_DESIGN.md](NETWORK_DESIGN.md) §1/§6.3 与 [DECISIONS.md](DECISIONS.md) D29/D30。

## 3. 完整流程（管线怎么走）

### 3.1 推理流程（给一张照片，渲染散景）

```
照片 I ──► ① Depth Anything V2-Small（冻结25M）──► 视差图 D（近大远小）
                                                    │
              ② 视差边缘吸附 snap_disparity_edges ◄──┘   （扫平深度网在物体边界的"软坡"）
                                                    │
        ③ 带符号 CoC：r = K·sign(δ)·max(|δ|−δ0, 0)   （δ0=对焦容差，保住主体）
                                                    │
        ④ 按视差分 K 层（tent 软权重）                │
                                                    ▼
   ⑤ 每层现场算 PSF（optics/）：该层离焦 + 像差向量 a ──► 该层的"虚化刷子"
                                                    │
   ⑥ 图像切 64px tile，每个 tile 用它所在 (像高H, 方位角) 的 PSF
      （PSF 按 +x 约定生成，按 tile 方位角旋转 → 猫眼/旋焦在画面边缘涌现）
                                                    │
   ⑦ 每层 FFT 卷积 → 从远到近 alpha 合成（遮挡）→ 乘回边角失光 T(H)
                                                    ▼
   ⑧ 边界细化网（v2 719k，唯一可训练）：整图单次前向（逐像素物理条件图，P1）
      修复深度边界瑕疵，平坦区透传物理结果（v1 的 512² 滑窗+Hann 已退役）
                                                    ▼
                                              浅景深输出 B
```

控制向量 **c = (对焦 d_f, 光圈 K, 像差 a)** 同时进入 ⑤（物理塑形 PSF）和
⑧（FiLM 条件注入细化网）。完整跑通这条链的脚本是
`python -m render.e2e_demo`（[render/e2e_demo.py](render/e2e_demo.py)），
它会输出"纯物理渲染 vs 细化后"的并排对照 + 误差图 + 边界放大窗。

### 3.2 训练数据怎么造（在线合成，`data/synth.py`）

物理渲染器**没有任何权重**，需要训练的只有边界细化网。训练数据**在线合成**，
每一步现做现用、永不重复：

1. 随机组装场景：真实照片背景（COCO train2017，118k 张池子随机裁块）
   + 程序化生成的透明前景（傅里叶轮廓 blob **或细薄结构：树枝/草丝/栅栏**，
   噪声纹理，**已知精确 alpha**）+ 随机亮斑；**关键补充（D23）**：以概率在背景侧
   插入**细薄层并强制对焦前景** → 它们重度虚化、GT 也虚化 = 教网络"背景细薄结构
   该保持虚化"，这是治"找补"的根因之药（细薄结构若只当近前景、对焦时 GT 清晰，
   网络会学成"细薄=该锐化"，在真实背景树枝上闯祸）；
2. 随机采样控制向量 c（各系数**独立**采样 → 不同效应天然不相关 = 解耦监督）；
3. **GT**：用已知 alpha 做**精确分层合成**（没有深度误差的完美答案）；
4. **网络输入**：单图 + **故意加扰动的视差**（边界形变/高斯模糊/低频噪声三连，
   模拟 DA V2 的真实误差形态）。

第 3、4 步的"GT 干净、输入脏"是防 sim-to-real 崩坏的关键（BokehMe 的核心技巧）：
细化网被迫学会"在深度不可靠的边界上修图"，这正是它在真实照片上的工作。

另一个省钱技巧（**视场相关监督**）：训练裁块只有 512²，把它放进一个"虚拟画幅"
里——每个样本随机采 (H_field, 方位角, 画幅大小) 决定这块在画幅哪个位置。v1 时
块内像高 H 当常数、用单个旋转 PSF 渲整块；**v2（P1）升级**：块内 H 跨度大时把 GT
分 2–3 个 H 子带分别渲染再 tent 混合，并配套逐像素条件图（§2.5）→ "条件图说 H 在变、
GT 也真的在变"，不再自相矛盾。猫眼/彗差/像散/LaCA 由此拿到廉价监督，(H, azimuth)
也写进 13 维 ctrl_vec 喂给网络。13 维的完整顺序（**改动必须同步 refine/network.py**）：

```
[d_f, K/100, W040, W131, W222, W220, n_blades/10,
 vignette_strength, vignette_radius, loca_R, laca_R, H_field, azimuth/π]
```

### 3.3 训练循环与看懂日志（`train/train.py`）

每个训练步：合成样本 → 扰动视差渲染出 B_phys（no_grad，跟推理同路径）→
细化网前向 → 四项损失反传：

```
loss = 边界加权 L1(B, GT)            ← 主目标（边界像素权重×5，否则梯度被平坦区稀释）
     + 0.1 · LPIPS(B, GT)            ← 感知损失（VGG 特征距离，治"糊但 L1 低"）
     + 0.5 · L1(B_neural, GT)        ← 神经分支直监（不经 mask 门控）
     + 0.2 · L1(m, m_gt)             ← 误差图显式监督，m_gt = clamp(10·|B_phys−GT|)
```

后两项是**血泪教训**（D19）：只用融合损失训练，误差图会"塌缩"——神经分支初始
略糊 → m>0 处损失更大 → 梯度把 m 压到 0 → 神经分支从此没梯度，网络退化成透传。
给误差图配显式 GT、给神经分支配直监，两条腿各自站稳，融合才有意义。

**看懂训练日志**（一行一个字段对照）：


## 4. 从来手法 vs 提案手法

### 4.1 现有方法版图（三条路线 + 一个旁支）

**路线 A：经典渲染 + 神经修补**（BokehMe CVPR22 / BokehMe++ TPAMI25 —— 本项目的骨架来源）
用理想圆盘/多边形核做散射式渲染，神经网络修边界。可控对焦和光圈，速度快。
**但 PSF 是"理想"的**——没有球差/彗差/像散，调不出镜头风格（BokehMe++ 加了猫眼，是个例外但仅此一项）。

**路线 B：端到端神经渲染**（Bokehlicious ICCV25；BokehDiff ICCV25、BokehFlow 2025.11、
Generative Refocusing 2025.12 等生成式方法）
网络隐式地学出"虚化"。真实感强，但 PSF 埋在权重里**不可解释、不可单独调**
（Bokehlicious 只有光圈一个旋钮；BokehDiff 基于 SDXL 级骨干；2025 下半年的
流匹配/扩散新作把控制做到了"焦点位置+模糊强度+文本"，但仍无一个触及像差系数）。

**路线 C：光学仿真**（The Visual Computer 2010/2012 —— 必须引用并划清界限的先行者）
用真实镜头处方做光线追踪，物理最忠实。但**不可微、无学习、需要完整镜头设计参数**，
每支镜头一套数据，没法做"从照片反推风格"。

**旁支 D：镜头变换**（NTIRE 2023 挑战：BokehOrNot、NAFBET 等）
"把 Sony 50mm 的焦外转换成 Canon 50mm 的"。条件是**镜头名 one-hot + 光圈数值**
——黑盒标签，不能连续插值、不能组合、不能反演。它是我们"物理系数条件"的
天然消融对照组（NETWORK_DESIGN P1c），其数据集 BETD 还是 RQ3 定量评测的钥匙。

### 4.2 对比表

| | A 经典+神经 | B 端到端神经 | C 光学仿真 | D 镜头变换 | **本项目** |
|---|---|---|---|---|---|
| 风格控制 | 光圈/对焦(+猫眼) | 光圈/焦点/文本 | 全部但需镜头处方 | 镜头标签离散切换 | **低维像差向量 a，连续可调** |
| 可解释性 | 核是理想形状 | ❌ 黑盒 | ✅ 物理 | ❌ 标签 | ✅ **每个旋钮有物理名字** |
| 可微性 | 部分 | ✅ | ❌ | ✅ | ✅ **全链路（含 PSF 生成）** |
| 镜头指纹反演 | ❌ | ❌ | ❌（正向仿真） | ❌ | ✅ **梯度下降反求系数** |
| 可训练参数 | ~数M | 大（SDXL级↑） | 0 | ~数十M | **0.68M（12GB 单卡）** |
| 深度输入 | 需要 | 部分不需要 | 需要场景3D | 不需要 | 需要（DA V2-Small 冻结） |

### 4.3 一句话总结异同

> 我们**继承**路线 A 的架构骨架（分层渲染 + 轻量细化网 + 深度扰动训练），
> **替换**它的心脏——把"理想圆盘核"换成"可微的像差化 pupil→PSF 物理引擎"（向路线 C 借物理、但做成可微可学习），
> 从而得到路线 B/D 给不了的**可解释、可解耦、可反演**的风格控制，
> 并坚决不跟路线 B 拼大模型真实感（差异化生存）。

2026-06 的完整文献坐标（条件注入机制演进线、撞车检查、我们的空位）
见 [NETWORK_DESIGN.md](NETWORK_DESIGN.md) §3 与 [references.md](references.md) 第六组。

## 5. 创新点（写论文时的 claim 清单）

1. **像差作为可控旋钮**：首个把 **12 种像差**（球差/**高阶球差**/彗差/像散/场曲/多边形光圈/
   口径蚀/**变迹STF**/LoCA/LaCA/**球色差**）做成 **连续、物理可解释、可解耦**控制接口的散景渲染
   框架；风格不是标签而是物理参数，组合效果（单侧亮边=亮环×猫眼、nisen=W040×W060）自动涌现。
   （文献铁证 D34：7 篇 SOTA 无一建模像差。）
2. **可微 pupil→PSF 引擎 + 镜头指纹**：`|FFT{P}|²` 全链路可微，同一套代码既支撑
   端到端训练、又支撑从真实样张**反求像差系数**（已验证：自标定误差 <1.1%，
   两阶段延拓协议见 D12）——路线 C 的物理 + 路线 B 的学习能力。
3. **轻量**：可训练参数只有边界细化网 0.68M（深度网冻结、渲染器无权重），
   12GB 消费级单卡可训练——与 SDXL 级扩散方法形成数量级对比。
4. **可控性的评测方法论**：效应签名矩阵（每个旋钮配一个专属可测签名，
   对角占优=解耦好）+ 指纹自标定闭环——给"可控性"这个模糊概念配上可复现的数字。
5. **（规划中，v2）renderer-informed PSF 描述子条件**：把渲染器对每个局部 PSF 的
   解析统计量（半径/拉伸/取向/环比/失光）做成逐像素条件图喂给细化网——
   散景文献无先例；同一套统计量身兼解耦评测指标、网络条件、指纹特征三职
   （"签名三位一体"，NETWORK_DESIGN §4.2/§7）。
6. **（规划中，M3 末）镜头指纹的定量评测**：用 BETD 双镜头数据做
   "标定 Sony → 渲染成 Canon → 对 GT 算指标"，把 RQ3 从定性复刻升格为可比数字。

诚实的自我定位（审稿人视角）：单看每个组件都有前作，**创新在组合与打通**
（物理可微化 + 解耦监督 + 指纹反演 + 轻量集成）。因此论文必须主动引用
Visual Computer 2010/2012 与盲像差估计文献并划清界限——这条线已写进 references.md。

## 8. 常见坑速查表（新人必读，全部实测踩过）

| # | 坑 | 一句话规避 | 出处 |
|---|---|---|---|
| 1 | 把深度当视差用 | CoC 公式里的 D 是**视差**（近大远小）；用任何深度源前先确认方向 | CLAUDE.md §12 |
| 2 | 球差/离焦符号配对搞反 | 亮边环在 **sign(W040)≠sign(W020)** 侧；验证肥皂泡/奶油前先确认离焦符号 | D13 |
| 3 | PSF 在离焦轴上插值 | 两个不同尺寸的环线性混合=双环鬼影；**永远现场算 PSF**，只有标量统计才能插值 | D9 |
| 4 | 旋转方向 90° bug | 验证旋转类效果不要用"顶边"这种退化位形，用四角点阵 | D13 |
| 5 | 误差图塌缩 | 融合门控必须配显式 mask 监督 + 神经分支直监，否则网络退化成透传 | D19 |
| 6 | 输出头严格零初始化 | W=0 会阻断流向 FiLM 的梯度，条件分支永远学不动；用 std=1e-4 近零初始化 | D18 |
| 7 | azimuth 范围错配 | 训练存的是 [0,2π)/π；`atan2` 返回负角必须先 `% 2π` | D20 |
| 8 | alpha 用单通道卷积 | alpha 与波长无关，必须用**均值核**卷，否则 LoCA 下边界彩色失配 | D17 |
| 9 | 主体被虚掉 | DA V2 把主体拉出视差跨度；上对焦容差 δ0，球差预设还要焦移补偿 | D14 |
| 10 | W020 超采样上限 | 光瞳网格 N=256 时 W020<32 waves（盘半径 ≲128px），超了 wrap-around 混叠（有 guard 告警） | CLAUDE.md §3 |
| 11 | 全图 L1 看不出网络收益 | >90% 像素是平坦区；**细化网收益一律看边界 L1**（m_gt>0.2 像素） | D19 |
| 12 | 训练/推理路径不一致 | 输入侧永远是"扰动/估计视差 → snap_edges → 渲染"，GT 侧才用精确 alpha | D18④ |

## 9. 术语小词典（按出现频率排序）

| 术语 | 一句话解释 |
|---|---|
| 视差 disparity | 1/深度的归一化版本，近大远小；DA V2 的输出，CoC 公式用它 |
| CoC | 弥散圆半径，`r=K·(D−d_f)`；带符号（正=前景虚化、负=背景虚化） |
| PSF | 点光源被镜头摊成的图案 = 散景盘的形状 |
| 光瞳 pupil | 镜头出瞳处的光场（振幅 A × 相位 e^{i2πW}），FFT 后得 PSF |
| 波前像差 W | 相位偏离理想球面波的量，单位 waves；W040=球差等 |
| 像高 H | 像素到画面中心的归一化距离（0=中心，1=对角顶点） |
| 方位角 azimuth | 像素相对画面中心的极角；猫眼/彗差的取向由它决定 |
| 口径蚀/猫眼 | 边缘视场光束被镜筒裁切 → 散景盘成柠檬形；T(H)=对应的能量损失 |
| LoCA / LaCA | 纵向/横向色差：前紫后绿 / 边缘紫绿镶边 |
| tent 权重 | 视差分层的三角形软权重（相邻层线性过渡，避免量化硬边） |
| alpha 合成 | 从远到近 `out = B + (1−A)·out` 的图层叠加（遮挡的来源） |
| FiLM | 用条件向量给每个通道做仿射调制 `h·(1+γ)+β` 的条件注入法 |
| 误差图融合 | `B = m·B_neural + (1−m)·B_phys`；m 是网络自己预测的"物理哪里错了" |
| LPIPS | 用预训练 VGG 特征算的感知距离；比 L1 更接近人眼对"糊"的判断 |
| 边界 L1 | 只在 m_gt>0.2 像素上算的 L1——细化网净收益的唯一可信口径 |
| 镜头指纹 | 对像差系数梯度下降、让渲染结果逼近某镜头样张 → 反求出该镜头的系数 |
| 在线合成 | 训练数据每步现场生成（背景裁块+程序化前景+随机系数），无离线数据集 |
