## D1 — 实现起点选 `optics/` PSF 生成器，而非先全量复现 M0 基线
- **日期**：2026-06-09
- **背景**：路线图 M0 写的是"先复现 DA V2 / BokehMe / 数据集"，但当时 GPU 不可用、环境未建。
- **决策**：第一块**自己写**的代码切在 `optics/` 像差化 PSF 生成器。
- **理由**：① 它是项目**真正的创新内核**，工程量最大、最该尽早试错；② 自包含、无权重、
  不依赖深度网/数据集；③ 产物（PSF 图）**可肉眼即时验证**，契合 CLAUDE.md「每模块配最小可视化」；
  ④ FFT 在 CPU 即可，**不被坏掉的 GPU 阻塞**。基线复现(M0)并行准备、互不挡路。
- **影响**：M1 提前破冰并已验证成功；M0 降为"下载+待冒烟测试"的轻任务。
- **可逆性**：高——基线随时可补。

## D2 — 深度骨干用 **Depth Anything V2-Small**，而非更新的 Depth Anything 3
- **日期**：2026-06-09
- **背景**：用户问是否该升级到 2025-11 发布的 DA3。联网调研了 DA3 的尺寸/许可/定位。
- **调研结论**：DA3（ByteDance-Seed）定位转向**多视图几何**，用 depth-ray 表示。
  Apache-2.0 的变体有 DA3-Small(80M)/Base(120M)/Mono-Large(350M)；Large/Giant 为 CC-BY-NC（非商用）。
- **决策**：**默认保持 DA V2-Small(25M)**；把 [depth/estimator.py](depth/estimator.py) 设计成**后端无关**，
  DA3 仅作为"边界质量成瓶颈时"的**消融/升级备选**。
- **理由**：① **轻量是项目卖点(RQ2)**，25M ≪ 80–350M；② V2 直接输出正方向相对视差，
  正好喂 CoC，DA3 的 depth-ray 还需转换+确认方向；③ DA3 的多视图/位姿优势对**单图散景无用**；
  ④ V2 是散景文献通用基线，对比更公平；⑤ DA3 还需 xformers，更重。
- **影响**：管线锁定 V2-Small；但封装留好了换后端的口子，对应 CLAUDE.md 第 9 节"深度消融"。
- **可逆性**：高——后端无关设计使换 DA3 不动主线代码。

## D5 — PSF 生成器**全程 torch、不写 numpy 临时版**
- **日期**：2026-06-09
- **背景**：曾考虑"先 numpy 验证物理、再移植 torch"。
- **决策**：直接用 torch 实现，device 无关（CPU 即可验证）。
- **理由**：**可微性是硬要求**（CLAUDE.md 第 12 节）——`|FFT{P}|²` 须对像差系数可微以支撑
  训练+指纹标定。torch 一步到位，避免写一份再扔。FFT 在 CPU 同样可跑，验证不被 GPU 阻塞。
- **影响**：optics 已在 CPU 验证物理与梯度（PROJECT_STATUS §6）；GPU 恢复后零改动可上 cuda。
- **可逆性**：高（但无理由回退）。

## D6 — 猫眼(口径蚀)**固定沿 +x 轴**计算，方位角自由度移出 PSF 字典
- **日期**：2026-06-09
- **背景**：口径蚀应指向像点径向方向，若按方位角建字典会多一维、显存翻倍。
- **决策**：PSF 字典里**固定让猫眼沿 +x 平移**；渲染时由 `render/` 按每像素方位角**旋转 PSF** 到正确朝向。
- **理由**：把方位角从字典解耦出去 → 字典只需按"像高 H"一维分桶，**省显存**（契合 12GB 约束）。
  猫眼几何当前用"双圆相交"近似（半径 R_v、圆心随 H 平移），与 BokehMe++ K0/K,z_l 同源，后续可标定。
- **影响**：`optics/pupil.py` 的 `vignette_mask` 按此实现；M2 渲染器需实现按方位角旋转取用。
- **可逆性**：中——若旋转开销/精度不理想，可改回二维字典。

## D7 — GPU 不可用判定为"**驱动版本错位、需重启**"，非硬件故障
- **日期**：2026-06-09
- **背景**：`nvidia-smi` 报 "Driver/library version mismatch"，torch 报 CUDA Error 804。
- **诊断**：运行中内核模块 `580.126.09`（旧）≠ 用户态库/磁盘模块 `580.159.03`（apt 已升级未重启）。
  显示器在 nvidia(card1) 上、Xorg+gnome 占用模块，且 sudo 需密码 → 热重载会崩桌面且无法免密。
- **决策**：判定为**版本错位**，修复方式为 `sudo reboot`（由用户执行）；不在线热重载。
- **理由**：重启后自动加载新模块、三层版本一致，最稳妥；热重载得停桌面、风险高、收益低。
- **影响**：M1 物理验证改用 CPU 完成，未受阻；GPU 相关冒烟测试推迟到重启后。
- **可逆性**：N/A（环境事实）。

## D8 — M2 渲染器路线：**分层 gather + tile 近似**，不从零复刻 Dr.Bokeh
- **日期**：2026-06-10
- **背景**：项目自审发现 M2 是最大可行性风险：D6 的"渲染时按像素方位角旋转 PSF"
  与任何高效卷积/scatter 实现都冲突——逐像素旋转 PSF 又慢又难写难调；
  且 Dr.Bokeh 的精确遮挡机制对初阶开发者从零复刻难度过高。
- **决策**：渲染器采用**分层 gather + tile 近似**：
  ① 按视差分 K 层；② 图像切 tile（如 64px），每个 tile 用其中心 (像高 H, 方位角)
  对应的 PSF（从 +x 约定的 PSF 旋转一次到该 tile 朝向）做 FFT 卷积；
  ③ 层间前后 alpha 合成 + 归一化处理遮挡。Dr.Bokeh 式精确遮挡留作改进项而非起点。
- **理由**：① 空变模糊的标准做法（refs 里 MPIB 同族），**可微、显存可控**；
  ② 旋转从"逐像素"降为"逐 tile"，D6 的字典省显存收益保住、开销可控；
  ③ 每层每 tile 都能单独可视化，契合"产出可验证"协作规范；
  ④ tile 内 PSF 视为常量的误差 ≪ 视场变化尺度（tile 64px vs 半幅 ~1000px）。
- **影响**：`render/renderer.py` 按此实现；CLAUDE.md 第 3 节增补"渲染器设计约定"。
- **可逆性**：中——接口不变的前提下可日后替换为 Dr.Bokeh 式精确渲染。

## D9 — PSF 字典降级为**推理缓存**；训练/标定按层**现场算 PSF**
- **日期**：2026-06-10
- **背景**：自审发现字典在离焦轴上做线性插值有原理性伪影：两个不同尺寸的
  环状 PSF（肥皂泡/洋葱圈）线性混合得到"双环鬼影"，不是中间尺寸的环——
  恰好伤害本项目最在意的风格效果。
- **决策**：训练与指纹标定时**不走字典插值**：渲染器按视差分层后每层离焦量确定，
  直接对这些确定值现场算 PSF（每前向约 层数×H桶×3 个 256² FFT，GPU 毫秒级）。
  字典 (`build_psf_dictionary`/`sample_psf`) 保留，定位为**推理时的加速缓存**
  （镜头系数固定的场景）。
- **理由**：① 彻底绕过插值伪影；② 对 a 的梯度链路更直接（无插值中介）；
  ③ 计算量实测可忽略，不破坏 12GB 约束。
- **影响**：psf.py 模块头与 docstring 已改写定位；M2 渲染器主路径不依赖 sample_psf。
- **可逆性**：高——两条路径并存，按场景选用。

## D10 — PSF 物理修正三件套：LaCA 改径向平移 / 保留口径蚀透过率 / 混叠 guard
- **日期**：2026-06-10
- **背景**：项目自审发现三处物理建模问题。
- **决策与理由**：
  ① **LaCA 从"径向缩放"改为"径向平移"**：横向色差的本质是各通道放大率不同 →
    离轴处各通道 PSF 中心彼此**错开**（平移 ∝ H），不是绕自身中心缩放
    （缩放更接近球色差，渲染不出真实的单侧紫/绿镶边）。实现沿 +x 平移
    `laca_rgb[c]·H` 像素，与 D6 猫眼约定一致、渲染时随方位角一起旋转；
    旧实现还有 bug：缩放未乘 H，画面中心也会被错误施加色差。
  ② **保留口径蚀相对透过率** `pupil.relative_transmission`：PSF 逐个归一化 sum=1
    丢掉了猫眼的"边角失光"分量；T(H)=ΣA²(含猫眼)/ΣA²(无猫眼) 存入字典，
    渲染合成时乘回。它也是解耦矩阵里猫眼的**专属签名**（相位像差不改变 T）。
  ③ **混叠/裁剪 guard**：相位欠采样上限 W020 < N/8 waves（N=256 → ≈32 waves），
    超限 PSF wrap-around 且被"裁剪+重归一化"无声掩盖。`pupil_to_psf` 现检查
    全图边界带能量（混叠）与裁剪丢失能量（>1% 告警）。
  附带：洋葱圈增加**相位调制选项** `onion_as_phase`（Sivokon-Thorpe 原机理为
  面形误差→相位涟漪，环对比度随离焦变化），默认仍为振幅版，留作消融。
- **影响**：`optics/aberrations.py`、`pupil.py`、`psf.py` 已改；
  `optics/calibrate.py`（离焦标定）与 `optics/decoupling.py`（解耦矩阵）两个
  验证脚本落地，结果见 PROJECT_STATUS §6。
- **可逆性**：高（①②③ 均有独立开关或可单独回退）。

## D11 — 评测协议明确"可控性指标的循环性"措辞；解耦度指标形式化
- **日期**：2026-06-10
- **背景**：自审指出：RQ1 可控性指标的 GT 由本项目自己的渲染器合成，度量的是
  "细化网对物理模型的忠实度"而非真实感——这样设计是合理的（可控性只能这样测），
  但若论文不主动说破会被审稿人视为循环论证。
- **决策**：① CLAUDE.md 第 9 节明确该措辞：可控性保真 = 对物理模型的忠实度，
  真实感单独由 RealBokeh/EBB! + user study 承担；② 解耦度指标落地为
  "效应签名矩阵"：每个旋钮配一个专属可测签名（球差→外环/内盘亮度比、
  彗差→x 向偏度、像散→log(σx/σy)、猫眼→相对透过率、洋葱圈→径向涟漪强度），
  逐旋钮扰动测全签名的带符号变化、按列归一化，对角占优=解耦好。
- **理由**：把模糊的"cross-talk 度量"变成可复现脚本（`optics/decoupling.py`），
  M1 即产出第一份证据（PSF 层面），M3 在图像层面复用同一套签名。
- **影响**：首次运行结果（对角 |·|=1.00/1.00/0.78/1.00/1.00）记入 PROJECT_STATUS §6；
  猫眼对偏度的 +0.31 串扰是真实物理耦合（猫眼+离焦本就移动散景球），论文如实报告。
- **可逆性**：高（签名定义可迭代）。

## D12 — 镜头指纹标定采用**两阶段延拓法**（低阶先行 → 联合微调），不做单阶段联合优化
- **日期**：2026-06-10
- **背景**：M2 渲染器全程可微后，用自标定实验验证 RQ3 链路：已知 c*=(d_f, K, W040)
  渲染目标图，从错误初值梯度下降反求。**两种朴素方案都掉进局部极小**：
  ① **单阶段联合优化**：优化器用"全局球差模糊 W040"冒充"深度相关离焦模糊 K"
    ——W040 跑到 −2.3、K 塌缩到下限 2.0（损失 0.039 平台）。
  ② **分阶段但 stage2 从错误初值重启**：stage1 在"无像差"假设下把 d_f 压到边界 0.02
    去模仿目标镜头的球差柔光（模型失配偏差）；stage2 若让 W040 从对抗性初值
    (−0.8) 出发且学习率压小，则被拽回坏盆地、走不回真值。
- **决策**：**两阶段延拓（continuation）**：
  stage1 冻结像差为 0，只拟合低阶 (d_f, K)（120 iters）；
  stage2 三参数全开、**W040 从 stage1 的值(0)延拓出发**、保持正常学习率（300 iters）。
- **结果**：d_f/K/W040 相对误差 **0.5%/0.4%/0.3%**，420 iters 共 11s（384px, RTX 5070）。
  stage2 轨迹显示 (d_f, W040) 沿耦合方向同步爬回真值——印证"先低阶后高阶"的设计。
- **影响**：`render/fingerprint_demo.py` 落地；M4 真实镜头标定沿用同一协议；
  日后扩参（vignette/彗差/色差）继续按"低阶→高阶"逐级延拓加入。
  这也是论文 method 的现成素材（损失面分析 + 标定协议）。
- **可逆性**：高（协议层面，可换多起点/多尺度损失等增强）。

## D13 — 用户审图发现的四个渲染问题：根因与修复（含一个 90° 旋转真 bug）
- **日期**：2026-06-10
- **背景**：用户人工检查 `outputs/render_test/` 指出四个问题。逐一诊断：
- **① 深度边界"台阶"→ 主体周围半虚化轮廓**：DA V2 在物体边界输出 10~20px 软坡，
  坡上像素落入中间视差层 → 渲染成一圈"既不实也不虚"的轮廓。
  **修复**：渲染前做**视差边缘吸附** `snap_disparity_edges`（局部 min/max 落差超阈值
  判为真边界 → 像素吸附到更近的一侧台面，迭代扫平；平缓渐变不受影响）。
  这是输入预处理，不在 c 的梯度链路上；训练时输入端同样适用。
- **② 猫眼方向错 90°（真 bug）**：`rotate_psf` 旋转符号反了——affine_grid 的矩阵
  作用在**采样坐标**上，M=R(+a) 时内容呈现为转 **−a**。
  **教训（重要）**：PSF 形状是"轴"(mod π)，上下左右四个正方向上 ±a 不可分辨——
  之前用顶边亮斑验证恰好是退化情形，误判为正确。**轴向旋转必须用对角位置验证**。
  修复后点阵图四角猫眼呈切向排列（正确旋焦）。
- **③ 六边形/洋葱圈看不清**：demo 对焦在主体(d_f≈0.24)而背景视差≈0 → 背景 CoC
  仅 ~10px，盘内结构不可见。**修复**：K 改为按"背景 CoC≈35px"反推（上限 105px），
  高光增益 4→8；另加合成"夜景灯点"展示面板 `demo_lights_grid.png`（盘结构的
  决定性展示，真实照片的灯往往太暗太小）。
- **④ soap_bubble/cream_soft "看反"**：不是预设错，是**验证用错了离焦符号**。
  物理：横向光线像差 ∝ 2·W020·ρ + 4·W040·ρ³，**亮边环出现在 sign(W040)≠sign(W020)
  的一侧**（边缘光线折返成焦散环），同号侧为奶油。本项目约定背景(D<d_f)→W020<0，
  预设命名按**背景侧**（经典 Trioplan 场景）定义 → soap_bubble=W040>0 **正确**；
  但旧 visualize 用 W020=+12（前景侧）核对，等于按反转后的形态验证了命名。
  **修复**：visualize 预设改在 W020=−12 验证并同时展示两侧（形态反转本身是
  标志特征）；decoupling 的测试离焦同步改为 −12，ring_ratio 签名符号转正。
  2026-06-09 的"7 效果涌现"验证记录中 soap/cream 两项按当时的图实为反相，以本条为准。
- **影响**：`render/renderer.py`（rotate_psf、snap_disparity_edges）、`render/demo.py`、
  `optics/visualize.py`、`optics/decoupling.py` 已改；全部产物重新生成并人工复核。
- **可逆性**：①③ 高（参数/预处理）；②④ 是正确性修复，不可回退。

## D14 — 渲染控制新增**对焦容差 δ0** 与**球差焦移补偿**（解决"主体被虚/虚化过量"）
- **日期**：2026-06-10
- **背景**：用户反馈 demo 虚化量过大且主体也被虚。诊断出两个独立原因：
  ① DA V2 的**相对视差**会把主体自身拉伸出 0.1~0.3 的跨度（相对深度归一化的固有
  现象，非误差），K 稍大主体内部的视差差异就被渲染成可见模糊；
  ② 球差预设（W040≠0）在 W020=0 焦面本就有满幅光晕——但真实摄影中球差伴随
  **焦移**，摄影师会重新对焦到"最佳焦面"，用离焦抵消部分球差。
- **决策**：`RenderControl` 新增两个控制：
  ① **focus_tolerance δ0**（视差半带宽）：r = K·sign(δ)·max(|δ|−δ0, 0)，景深带内
  CoC=0。demo 中 δ0 取主体区域视差 p15~p85 半宽（实测 ≈0.10）。
  ② **balance_spherical_focus**（默认开）：所有层叠加恒定 W020 偏移 −W040
  （波前平衡：ρ⁴−ρ² 是平衡球差，W020=−W040 时 RMS 最小，等价于"对焦到最佳焦面"）。
  另把 demo 的虚化目标调温和（背景有效 CoC≈22px 目标、全图上限 60px）。
- **结果**：soap_bubble/cream_soft 的主体核心收紧（眼镜/纹理清晰，残留轻微柔光
  =球差镜头实拍观感），背景肥皂泡/奶油形态保留；指纹自标定回归不受影响
  （0.8%/0.0%/0.2%）。
- **影响**：两个控制都在 c 的可微链路上（relu 次梯度 / −W040 线性项）；
  训练（synth 默认 δ0=0）与指纹标定的前向模型保持一致。
  开放问题：synth 是否随机采样 δ0 并纳入 ctrl_vec（现 14 维），接细化网时定。
- **可逆性**：高（均为可选项，默认行为有 flag 控制）。

## D16 — 散景盘内能量不均的根因与修复：**光谱平均**（±6% 带宽 3 子波长）
- **日期**：2026-06-10
- **背景**：用户指出渲染的光斑内部能量分布不均匀（盘内环纹/网纹）。
- **诊断**：这是**单色相干仿真的菲涅耳衍射环**——纯波动光学下离焦 PSF 盘内
  天然有高对比干涉环；但真实相机是宽光谱 + 部分相干 + 传感器积分，
  实拍散景盘接近均匀。即"物理没错，但少了一层真实成像的物理"。
- **决策**：`rgb_psf` 默认开启**光谱平均**：每通道在 ±6% 带宽（≈相机色滤片有效宽度）
  取 3 个子波长加权平均（0.25/0.5/0.25）。实现上波前 W 以中心波长的 waves 计量、
  相位 ∝ 1/λ → 子波长等价于 `complex_pupil(phase_scale=λc/λj)` 缩放相位。
  环纹位置随波长移动 → 平均后互相抵消，盘面均匀；盘的尺寸/形状/边缘特征
  （亮边、猫眼、多边形、角点）保留。
- **结果**：A/B 对照 `outputs/psf_test/spectral_ab.png`——单色版盘内环纹/网纹明显，
  光谱平均后盘面均匀。全链路回归：标定斜率不变(3.93)、解耦对角占优不变、
  指纹自标定 1.1%/0.0%/0.1%、demo 灯点面板盘面均匀。
- **影响**：PSF 成本 3×（每 PSF 9 个 FFT），渲染耗时仍在 0.1~0.2s/张量级；
  `spectral=False` 可退回单色（调试/消融用）。
- **可逆性**：高（开关 + 带宽/采样数都可调）。

## D18 — 边界细化网设计：残差式 ARNet+IUNet 轻量版 + FiLM 条件 + 误差图融合
- **日期**：2026-06-11
- **背景**：M2 后半接入唯一可训练模块（refine/network.py）。BokehMe 的 ARNet 是
  "从 (图, 离焦图) 从零神经渲染散景"，本项目的物理渲染器已把像差风格做对，
  网络职责收窄为"修深度边界瑕疵"——结构据此改造。
- **① 残差式而非从零渲染**：ARNet 预测的是相对 B_phys 的残差，IUNet 预测的是
  相对上采样粗结果的残差；最终按误差图融合 B = m·B_neural + (1−m)·B_phys。
  理由：物理层的像差风格（肥皂泡/猫眼/色差）不该被网络重绘；小容量网络
  (684k 参数) 学局部修复远比学全图渲染可靠，可控性也不会被网络"洗掉"。
- **② FiLM 条件注入**：13 维 ctrl_vec → 2 层 MLP 共享编码 → 每个残差块一个
  线性头出逐通道 (γ,β)，h←h·(1+γ)+β。渲染器与细化网共享同一个 c
  （CLAUDE.md 第 2 节的一致性要求）。另把 r̃=K·(D̃−d_f)/128 作为输入通道
  （"离焦提示"），物理一致、零成本——依赖 ctrl_vec[0]/[1] 索引，synth 与
  network 两处注释互指。
- **③ 近零初始化，不能严格置零（踩坑）**：输出头若 W=0，d(head)/d(h)=W=0 会把
  流向 FiLM/cond 的梯度整体阻断（冒烟测试实测 ctrl_vec 梯度=0），条件分支
  永远学不动。改为 std=1e-4 正态初始化：起点仍≈透传物理渲染
  （|B−B_phys|=4e-4），梯度全通。
- **④ 训练输入侧与推理严格同路径**：扰动视差 → snap_disparity_edges →
  render_field_patch（单 PSF 视场近似，与 GT 的 composite_blurred_layers
  共享 (H_field, azimuth)，且同样乘 T(H) 保持亮度口径一致）。渲染在 no_grad
  下进行——训练时 c 是给定值，不需要梯度过渲染器（指纹标定才需要）。
- **规模**：ARNet 96ch×3 块 @1/4 分辨率 + IUNet 48ch×2 块 @全分辨率，
  共 684,035 参数（上限 2M 的 34%）；512²/batch4 实测 0.45s/it、峰值 5.7GB。
- **可逆性**：通道数/块数是构造参数，容量不够时可上探（2M 内余量 3 倍）。

## D19 — 误差图融合的"塌缩"问题：三项损失（显式 mask 监督 + 神经分支直监 + 边界加权）
- **日期**：2026-06-11
- **现象（首次 1500 步训练实测）**：只用融合输出的 L1+LPIPS 训练时，误差图 m
  全图→0，网络退化成纯透传（L1(B)≡L1(B_phys) 至小数点后 4 位，最终可视化
  中 |B−GT| 与 |B_phys−GT| 完全相同）。
- **根因（门控的先有鸡还是先有蛋）**：神经分支初始略糊（粗分支经过一次下/上
  采样）→ m>0 处损失更大 → 梯度把 m 压到 0 → m=0 后 ∂L/∂B_neural·m≈0，
  神经分支再也学不动。
- **对策（BokehMe 同款思路，三项加在 train/train.py）**：
  ① **误差图显式监督**：m_gt = clamp(10·|B_phys−GT|, 0, 1)（物理渲染的真实
     误差，phys/GT 都无梯度，是常量目标），L1(m, m_gt)，λm=0.2；
  ② **神经分支直接监督**：L1(B_neural, GT) 不经 mask 门控，λn=0.5——
     保证 m≈0 处神经分支照样有梯度；
  ③ **边界加权重建**：逐像素权重 1+4·m_gt（边界高误差区只占 ~8% 像素，
     不加权时边界梯度被平坦区稀释 ~12 倍），除以权重均值保持量纲。
- **验证**：加 ①② 后重训 1500 步，m 不再塌缩且精确点亮在物体边界
  （outputs/train_test2/viz_it001500.png）；但边界区域 L1 尚无净收益
  （0.0499→0.0498）——1500 步太短 + 梯度稀释，③ 即为此而加。
  长训练（10k 步）的定量结果记入 PROJECT_STATUS §6。
- **新增评测口径**：`evaluate_boundary`（train/train.py）——只统计 m_gt>0.2
  像素的 L1，phys vs fused 对照。全图 L1 被 >90% 平坦区主导，对"修边界"
  这个任务不灵敏，**细化网的净收益一律看边界 L1**。
- **可逆性**：λn/λm/边界权重在 configs/base.yaml train 节，可调可关。

## D20 — 端到端推理：细化网用 512² 滑窗 + 窗心视场条件 + Hann 融合（不整图单次前向）
- **日期**：2026-06-11
- **问题**：细化网在 512² 裁块上训练，且每个样本整块共享一个视场位置
  (H_field, azimuth)（synth 的单 PSF 视场近似）。真实整图推理时这两维怎么给？
  整图单次前向只能喂一个 (H, azimuth)，与"画幅各处视场不同"矛盾。
- **决策**（render/e2e_demo.py `refine_full_image`）：512px 窗口、256px 步长
  滑窗推理；每窗用**窗口中心**的 (H, azimuth) 拼 ctrl_vec（像高/方位角几何与
  render_tiled 完全同约定），各窗输出 Hann² 窗加权融合。
  - 与训练分布最贴近：窗口尺寸=训练裁块尺寸，视场条件=训练的"整块共享"口径；
    与渲染器的 tile 近似（D8）哲学一致。
  - 代价可忽略：0.7M 网络一窗 <10ms，1024 宽整图 ~35 窗共 ~0.2s。
  - Hann 窗边缘值 clamp 1e-3（四角只有单窗覆盖，权重恰为 0 会 0/0）。
- **易错点（实测踩坑）**：synth 的 azimuth ∈ [0,2π)、ctrl_vec 存 azimuth/π ∈
  [0,2)，而 math.atan2 返回 [-π,π]——必须先 `% 2π` 再除 π，否则负角超出训练分布。
- **验证**：512² 合成样本上滑窗 vs 直接前向，边界 L1 0.0353→0.0314（滑窗）
  vs 0.0312（直接），收益保留；|diff|max≈0.011 来自窗心 H 与样本 H_field 不同，符合预期。
- **已知现象（待 50k 正式权重复核）**：train_run1 的 10k 权重（25 张占位背景训练）
  在真实照片上把虚化前景的细枝纹理"找补"回锐利——背景池过小的过拟合，
  这正是换 COCO 118k 背景跑正式训练的动机；机制层面滑窗管线本身已验证无误。
- **可逆性**：patch/stride 是函数参数；若 50k 权重下整图单前向（H=0）质量无差，
  可简化回单前向。

    真实细枝对网络而言"处处是边界"，按训练学到的边界策略从全焦输入抄回锐利纹理；
  ② mask 监督只教了"哪里该亮"（m_gt=clamp(10·|B_phys−GT|)），没有任何机制
    约束它在真实纹理上的误触发——合成域里平坦区 phys≈GT，网络从未见过
    "纹理丰富但物理渲染正确"的负样本形态。
- **决策（三联对策，随 P1 一次重训，不单独烧训练）**：
  ① **细薄前景**（data/synth.py `_random_thin_alpha`）：程序化树枝/草丝/栅栏
    （宽 1–6px，cv2 折线/坐标场软方波），p_thin_fg=0.5 与 blob 混采——
    "细薄物保持虚化"由精确 GT 直接监督。副产物：1–2px 细丝 alpha<0.5 不进
    视差 GT，天然模拟 DA V2 漏检细结构的真实失败模式；
  ② **边界带门控**（refine/conditioning.boundary_band）：m ← m·band，
    band = 视差边缘（5×5 极差>0.04）扩张"图内最大 |CoC|+8px"（渗色传播距离的
    物理上限）再软化。带外神经分支**结构性失效**——红线（network 不得画风格）
    从损失约束升格为结构约束。mask 监督目标同乘 band（带外目标不可优化，
    徒增噪声）。代价（接受）：视差完全漏检的细丝带外无法修复——那属于
    "物理不可知"范畴，宁可保留物理结果也不开找补的口子（P3b 的领域）；
  ③ P1b 描述子图本身对症：网络确知"此处核半径 30px"，条件上抑制锐化倾向。
- **P1 实施细节中的新决策**：
  - 取向通道不用裸 sin2φ/cos2φ，改用**各向异性张量分量** (sxx−syy, 2sxy)/trace
    ——幅度=拉伸强度、盘各向同性时自动→0，规避"圆盘取向无定义"的噪声；
    按像素方位角旋 2·az（轴量 mod π，跨画幅中心 ±π 翻转无感）。
  - 描述子表走**单色** PSF（与 D11 解耦签名同口径，1 FFT/项 vs rgb_psf 的 9）。
  - H 子带 GT：块内 H 跨度 <0.12 单带（条件图同步退化为常数，防"条件说变、
    GT 没变"自相矛盾）、<0.30 两带、否则三带；子带混合的是**渲染结果图**，
    不是 PSF（D9 禁区不触）。输入侧 render_field_patch 用同一组 (centers, weights)。
  - 虚拟画幅几何用**精确极坐标**（非径向线性近似），跨中心裁块也正确。
- **代价实测**：参数 684k→719k（+35k）；训练 0.42→0.86s/it（子带 GT 为主），
  50k ≈ 12h；显存 5.8GB 不变。e2e 整图单前向 0.13s/张@1024×1280（滑窗退役）。
- **可逆性**：configs/base.yaml refine 节三开关（spatial_cond/psf_desc/
  mask_edge_gate）全关 = 复现 v1；ckpt 存 extra_cond_ch 标记，e2e 自动选路径。
