## D54 — tile 渲染路径能量诊断：PSF 空变的固有能量误差（已知限制）+ 归一化口径改进

- **背景（2026-06-23，用户反馈）**：用户观察"有的像差虚化后整体亮度与别的明显不同，是否正确"。
  系统排查发现亮度差异有三个来源，其中一个是真实缺陷。
- **三个亮度差异来源（点光源能量守恒实测，单点 RGB 总能量应=3.0）**：
  1. **显示归一化（主因，非物理）**：验证脚本 `normalize_for_display` 每图除以自身 max → peak/mean
     不同的像差整体明暗不同（ideal 1.07 平顶=亮；apod 4.0 中心亮边暗=整体暗）。是【展示方式】非物理。
  2. **物理正确的差异**：散景盘越大每像素越暗（能量守恒：under 盘大→暗 / over 盘小→亮）；猫眼 T(H)
     口径蚀【真的】让角灯变暗（实测总能量 2.59<3）；变迹边缘吸收。这些亮度差是【对的】。
  3. **tile 路径能量误差（真实缺陷）**：视场相关像差走 tile，实测点光源总能量 ideal +3%、场曲 +6%、
     彗差 +9.5%（>3.0 偏亮），误差集中在盘边缘。
- **tile 误差根因（逐项排除后定位）**：global 与单 tile(=整图)均守恒 3.000；多 tile 不守恒。排除了
  halo（33→120 无变化）、overlap（no-overlap 同样不守恒）、out/acc 归一化（子图中心区 out/acc 与
  全图逐像素一致）。**根因=tile gather 近似的固有性**：点光源散景盘横跨多 tile 时，盘的不同部分被
  各 tile 各自 (H桶,方位角) 的 PSF 渲染再拼接，块间 PSF 不同（hb 改盘大小、azim 改朝向）→ 拼接盘非
  任何单一 PSF 的完整盘 → 能量偏离。**误差随 PSF 不对称性单调增大（圆对称 ideal 3% < 像散 5% <
  场曲 6% < 彗差 9.5%）正是此机理的指纹。**
- **决策**：
  1. **接受为已知限制**，不重构 render_tiled（根治需 scatter 或空变卷积，工程量大/风险高）。**关键：
     只影响 e2e 整图推理的视觉亮度，不影响训练/科学结论**——训练走 `render_field_patch`（单 PSF 整块=
     与 global 同口径、守恒），合成 GT 走 `composite_blurred_layers`（同样单 PSF/块、守恒）。真实照片
     连续散景相邻盘重叠、误差被平均，远不如孤立点光源显著。
  2. **归一化口径改进（保留）**：`_composite_tile` 返回未归一化 (out, acc)，归一化移到全图做一次
     （`out_total/acc_total`），使重叠区按 alpha 加权混合、与 global 同口径。注：此改进让归一化更一致，
     但【不能】修复上述 PSF 空变误差（根因不在归一化，实测对 ideal 无改变）。
  3. **显示改进建议**：验证 montage 可加"统一亮度 scale"选项，让相对亮度可比（展示物理差异：盘大小/
     失光），与"各图独立归一化看形态"互补。
- **可逆性**：高。归一化口径改动局部（_composite_tile + render_tiled 循环）；本条诊断为日后 scatter
  重构的依据。

## D53 — balance 焦移补偿扩展到高阶球差 W060 + 像差能量分布系统评估（修 LoCA 夸张 / W060 前后不对称）

- **决策（2026-06-23，用户反馈驱动）**：① **球差焦移补偿 balance 扩展到 W060**（最小弥散圆系数
  1.5；原仅补 W040 系数 1.3）；② **nisen W060 回调 2.0→1.3**（撤 D52 增量）；③ **LoCA 验证展示
  幅度 1.5→0.6**（`sample_random` 的 0~0.4 范围本就温和、不变）。借此对 13 类像差做能量分布系统评估。
- **背景**：用户指出 LoCA 前后离焦图太夸张、W060(nisen)前后焦弥散圆大小不一致，并要求评估能量分布。
- **能量分布评估结论（全部实测，`outputs/aberration_grid/` 诊断图）**：
  - **能量守恒**：所有像差所有 RGB 通道 PSF sum=1.000，无能量泄漏 ✓。
  - **前后盘对称**：ideal/多边形/变迹/LoCA 前后对称(比 1.00~1.01)；球差类前后【不对称】是物理标志
    （under 0.81 / over 1.23，±20%，合理）；**nisen 修前 fg/bg=2.53(W060=2.0)/1.88(1.3) 异常大**。
  - **能量集中度 peak/mean**：ideal 1.07(平顶)、球差 2.0~2.3(焦散环/亮心)、nisen 3~4(尖锐亮环)、
    变迹 4.0(中心亮，a=2.5 偏强但属变迹物理)、色差 1.2~1.6——均在合理物理范围，**无非物理奇点**
    （几何散射 geom splat 的焦散环无单像素爆点）。
- **三处修正与理由**：
  1. **balance 补 W060（核心，`render/renderer.py:_layer_psf` + halo 估算 `_extent_at_disp`）**：
     最小弥散圆应补偿【所有】球差阶——各阶系数是与系数值无关的物理常数（数值实测：W040(ρ⁴)→1.3、
     W060(ρ⁶)→1.5）。**根因诊断**：nisen 前后盘 2.53× 不对称【主要是 balance 只补 W040 的人为加剧】
     而非纯物理——balance 偏移只含 W040 → 前后 W020 不对称、被 W060 高次项放大；补上 W060 后焦内残余
     9.7→3px、前后比 1.88→1.26（降到与初级球差相当，球差固有的前后异形本就小）。
  2. **nisen W060 2.0→1.3**：D52 增到 2.0 求二线分明，但中间平台仅 0.858→0.839(微小改善)却显著加剧
     前后不对称；balance 补 W060 后 1.3 已二线清晰 + 前后比 1.26，故回 1.3。**修正 D52 的 nisen 取值。**
  3. **LoCA 1.5→0.6**：±1.5 各通道盘半径差 ~12px → 整盘染绿/品红(夸张)；±0.6 差 ~5px → 盘【边】细
     绿/品红镶边、盘心近中性(真实 LoCA bokeh fringing)；`sample_random` 的 0~0.4(差 ≤3px) 本就真实。
- **影响**：① 所有含 W060 的渲染焦内更锐、前后更对称（训练 `sample_random(extended=False)` 不采 W060，
  **训练分布不变**）；② 球差 under/over(W060=0) 完全不受 balance W060 影响(1.5·0=0)、形态不变（回归验证
  soap/cream/nisen 渲染均有限值）；③ LoCA 视觉更接近真实。物理模型/能量守恒/解耦/红线/轻量约束不变。
- **可逆性**：高。balance W060 系数、nisen/LoCA 取值均独立可调；balance 仍受 `balance_spherical_focus`
  总开关控（关闭则回傍轴焦点）。

## D52 — 基于 7×7 点光源验证的像差真实性评估与三处修正（LoCA 色相轴 / tile 接缝 / nisen）

- **决策（2026-06-23，用户驱动）**：新建 `render/aberration_grid.py` 点光源验证工具（7×7 点阵 ×
  背景/焦平面/前景三工况 × 各像差），逐一肉眼+定量评估渲染真实性。评估确认**绝大多数像差物理正确**，
  仅发现并修复三处与现实的出入：① **LoCA 改绿-品红轴**、② **tile 接缝消除**、③ **nisen 二线优化**。
- **背景**：用户提三问——彗差为何不是十字、各像差弥散圆大小不一、球差合焦糊。系统评估后：
  - **确认正确（无出入）**：球差形态反转(soap 边/心=3.59、cream 心/边=1.51)、彗差彗星状且径向一致
    不随离焦翻转、**像散长轴背景切向/前景径向(翻转90°=子午弧矢焦互换)**、场曲角灯随 H² 离焦、
    猫眼橄榄长轴切向(旋焦来源)、多边形+星芒联动、变迹高斯渐隐、LaCA 红外蓝内径向、球色差盘边镶边。
  - **澄清用户三问**：十字是【多边形光阑衍射星芒】非彗差（已 D 前置说明，彗差恒彗星状）；弥散圆大小
    不一是物理（球差前后焦盘本就不对称 27/51px、是球差定义性标志，不归一化）；球差合焦糊是早期
    `balance_spherical_focus=False`(对焦傍轴焦点)所致，开启 balance(对焦最小弥散圆)后焦内只剩轻 glow。
- **三处修正与理由**：
  1. **LoCA 红-蓝轴 → 绿-品红轴**（`optics/aberrations.py`）：旧 `sample_random` 实现 `loca_rgb=(R+,0,B−)`
     得"焦后红心蓝边"，是简单消色差的单调色散；但 LoCA 字段注释本就写"R、B 相对 G 反向"——**实现与
     注释意图不符**。真实摄影最标志性的 LoCA bokeh fringing 是**焦后绿/焦前品红**（复消色差残余二级
     光谱，G 与 R/B 分离）。改 `loca_rgb=(+a,−a,+a)`（R/B 同向、G 反向）→ 验证图焦后绿边+品红心、
     焦前翻转。**ctrl_vec 兼容**：仍用 `loca_rgb[0]=a` 作 LoCA 强度标量，但语义变（需细化网重训生效，
     正合"重训待核心定稿"）。LaCA 保持红-蓝单调轴（放大率色差，单调正确）。
  2. **tile 接缝消除**（`render/renderer.py:render_tiled`）：散景盘横跨 tile 边界时，相邻 tile 用各自
     (H,方位角) 的 PSF **硬拼接** → 盘内可见十字缝/缺口（中心灯恰在 512=8×64 边界处最明显；global
     路径无此问题，证明是 tile 离散化伪影非物理）。改为 tile **50% 重叠 + 2D Hann 窗加权累加 / 除权重和**
     → 不同 PSF 在重叠区平滑过渡、盘内无缝。代价 tile 数约 4×（验证脚本 13s→26s），仅影响 e2e 整图
     推理；**训练用 `render_field_patch` 单 PSF 整块、不走 tile，不受影响**。
  3. **nisen W060 1.3→2.0 + 修正"中途环"说法**（`optics/aberrations.py` PRESETS）：评估初稿误判
     "nisen 应是盘内中途亮环"；扫描证明**大离焦散景盘下几何焦散环必在外缘**（各 W040/W060 组合峰位
     均 0.9~0.97·R，离焦项主导）——"中途环"仅中等离焦/近合焦时可能。nisen 的真实判据是**二线性**
     （双线测试：一条亮线被尖锐边环描成两条+中间暗，已成立）。W060 增大让暗心更暗、二线更分明
     （中间平台 0.858→0.839）。**撤回评估初稿的"中途环"不准确表述。**
- **影响**：① LoCA 视觉更接近真实、训练分布改变(待重训)；② e2e/swirl 等 tile 渲染盘内无缝、略慢；
  ③ nisen 二线略分明。渲染物理模型/12 种像差/解耦/红线/轻量约束均不变。验证产物见
  `outputs/aberration_grid/`（montage + 各 strip + 诊断图 `_*.png`）。
- **可逆性**：高。LoCA 改 `loca_rgb` 取值约定(git 可查)；tile overlap 是 render_tiled 内部改动、
  可退回硬拼接；nisen 一行参数。三者互相独立。

## D49 — 重大路线决策：镜头指纹反演(RQ3)降级为 future work，主线聚焦"像差可控的轻量化单图散景渲染"

- **决策（用户拍板，2026-06-22）**：把 **RQ3（镜头指纹反演 / lens fingerprint）从硬约束降级为
  future work**——主线投稿后再作为新阶段（记为 **P-FP**）攻坚。当前研究主攻 **RQ1 可控性 + RQ2
  轻量性**：各类光学像差可控的轻量级单图景深（散景）渲染。可微性、红线、评测、路线图据此重新校准。
- **背景**：评估发现 RQ3 的真实镜头反演是 **sim-to-real 风险最高、且最未验证**的一环——单光瞳
  多项式波前能否拟合真实多片镜组的空变像差，目前**只在自合成目标上验证过（=循环 / inverse crime）**，
  真实样张反演一次都没做。把它放在主线里会让确定能交付的工作被一个高不确定性目标拖累。
- **理由**：
  1. **RQ1/RQ2 是确定能交付的硬数据**（可控性保真 + 图像层解耦矩阵 + 轻量效率对比），完全不依赖真实反演；
  2. 指纹反演的科学风险（单波前保真度）**独立于主线**，可在主线稳定后单独攻坚，互不阻塞；
  3. **差异化叙事不依赖指纹也成立**——"控制虚成什么风格 + 物理可解释 + 轻量"三支柱靠 RQ1/RQ2 即可立论；
     指纹是锦上添花的应用，不是立论根基。项目标题本就是"像差可控的轻量级散景渲染"，未含指纹。
- **影响（逐项校准）**：
  - **可微性（CLAUDE.md §12）**：从"硬要求"降为"**保留的设计属性**"。当前训练本就在 `no_grad` 下跑物理
    渲染（PROJECT_STATUS §3），可微性此刻只服务（已降级的）指纹；但**保留它成本极低、且为 future 留门**，
    故不拆除，只是不再作为 gating 每次改动的硬约束。引入不可微操作不再一票否决，但需注明、且默认保守。
  - **红线（NETWORK_DESIGN §1）**：**保持不变，且变得更核心**。红线首要理由从"指纹需要"切换为
    "**RQ1 可控性需要**"——若风格部分来自网络，像差系数就不再真正控制风格，可控性立论即塌。**降级
    RQ3 ≠ 放松红线。**
  - **评测（CLAUDE.md §9）**：指纹协议(5)与(1)(2)中"对真实镜头"的部分降为 future/P-FP；主线评测=
    可控性保真(1) + 图像层解耦(2) + 真实感(3,降权) + 轻量效率(4) + 消融(6)。
  - **路线图（CLAUDE.md §10）**：M4 的"镜头指纹 demo"移出主线收官，改为**主线投稿后的新阶段 P-FP**。
  - **数据/参考（references.md 第五/七组）**：BETD 指纹定量、Eboli/CircleFlow/OmniLens++ 等盲反演
    related work、特色镜头样张，统一标注为 **P-FP 阶段所需**，主线不阻塞、不必现在接入。
  - **风险（CLAUDE.md §11）**：sim-to-real 的"指纹锚定"对策移到 P-FP；主线 sim-to-real 靠真实微调 +
    感知损失 + 扰动建模。
- **不变项**：像差物理模型、12 种像差、解耦矩阵、渲染器、**细化网红线**、轻量约束、范围边界
  （鬼影/眩光仍排除）、`render/fingerprint_demo.py`（保留，随时可重启为 P-FP）。
- **可逆性**：高——纯优先级/叙事调整，物理模型与代码不动；任何时候可把 RQ3 重新升格为主线。

## D50 — 网络职责再界定（基于 2026-06-22 路线评估对话）：细化网"必要但收窄"，深度/遮挡分工外移

- **决策**：明确细化网的**真正职责 = 合成几何不可知的边界内容**（被遮区域 disocclusion 外观、过渡带
  颜色）+ **对不完美深度鲁棒**；它**不**承担：修全局深度错、修渲染阶段遮挡、画风格。配套工程调整
  写入 **NETWORK_DESIGN §11**，标注"评估结论/待验证"，不立即重训。
- **背景**：连续多轮压力测试（要不要网络 / 前移还是后置 / 能否用网络做深度重映射）的收敛结论。
- **理由（逐条，详见 NETWORK_DESIGN §11）**：
  1. **细化网必要**（沿用 BokehMe 立论：对 imperfect disparity 鲁棒）；但 train_run6 合成边界 L1
     −0.0033(~5%) 是在"深度近乎完美"的合成集上测的，**低估了它在真实烂深度上的价值**——判据要换到
     真实图失败模式，不能只看合成边界 L1。
  2. **深度误差分两种**：局部边界错（网络 RGB 引导可修）vs 全局结构错（脸-树 merge / 错序）。**全局错
     网络修不了**——修它 = 造一个比 DA V2 强的单目模型，0.7M 头做不到。
  3. **位置 = 后置、输出几何**：保持在渲染器之后（需看到 B_phys 才知道修什么）、输出 matte/几何而非
     颜色（守红线）。**不**做"前移成可训练深度修正头"——既进轻量预算又撞"打败 DA V2"墙；"前移"仅作
     可选的 RGB 引导边缘锐化，**替代不了后置网**。
  4. **不 overload**：全局深度错 → **深度侧冻结先验**（DA V2-Large 已用；人像可选冻结 matting/分割
     先验，同 DA V2 口径、不算可训练参数、不破轻量，D25 先例）或记为已知限制；渲染阶段遮挡（色渗 /
     tent 漏光，D34）→ **渲染器内部 Dr.Bokeh 式层内/层间 alpha** 解决，非网络的活。
  5. **训练/真实失败模式失配（高 ROI 待办）**：`data/synth.py` 当前只对视差加**局部扰动**（边界形变/
     模糊/低频噪声），而真实 DA V2 错是**全局**（merge/错序）→ 网络没见过它要修的东西。**给合成扰动加
     全局/结构性扰动**（层间 merge / 深度压缩 / 错序），比换骨干、加容量 ROI 高。
  6. **可训练网络始终一个**：冻结先验（DA V2、可选 matting）不算贡献参数；**严禁训练自己的深度头**。
  7. **前景/背景虚化尺度**：不追深度重映射网络（D39 已撤回 γ）；靠 d_f/K + 上限控制（相对模糊，
     BokehMe 传统），可选冻结 metric depth 先验给物理正确比例；前/背景虚化量可作**明确标注非物理**的
     审美旋钮。
- **影响**：本条是**设计指引**，不改既有代码；落地顺序见 NETWORK_DESIGN §11（先做 with/without-net
  消融 + 合成扰动升级，再决定是否上 Dr.Bokeh 遮挡 / matting 先验）。
- **可逆性**：高——均为待验证建议，每项独立可回滚。

## D48 — cream 改用【轻球差 + 变迹】：修正"亮心没虚化"（球差柔化 ≠ 变迹虚化）

- **决策**：`cream_soft` 从 D47 的 `W040=−2.0 + W060=−0.8`（纯强欠矫正球差）改为
  `W040=−1.0 + apodization=1.5`（轻球差 + 变迹）。
- **背景**：用户反馈"cream 的 PSF 中间亮周围暗 → 生成图像没虚化一样、只是变糊了"。
- **根因（机制辨析）**：纯欠矫正球差的 PSF 是【圆顶/亮心】——卷积时中心权重高 → **保留焦外结构**
  （Helios 式【柔化】，非虚化）。实测 cream 背景枯枝隐约可见（vs ideal 完全化开），中心峰值/盘均值 1.24。
  这是欠矫正球差的物理本质（亮心 = 保留原结构），不是 bug，但与用户要的"虚化感"相反。
- **解法**：真正的奶油【虚化】靠【变迹 apodization】（Sony STF 机制：光瞳振幅径向衰减 exp(−aρ²)
  → 柔边【填充盘】= 均匀虚化 + 柔边无硬边），而非欠矫正球差（相位/亮心）。cream = 轻球差（−1.0 给一点
  亮心暖调）+ 变迹（1.5 给柔边填充虚化）。**三得**：柔虚化（背景化开）+ 略亮心（奶油感）+ 焦内更锐
  （变迹对焦内影响远小于强球差，顺带修了 D47 的 cream 焦内软）。
- **验证**：real21 cream 背景柔和化开（真虚化）+ 焦内女生明显更清晰；night 盘=柔边填充盘
  （`review_0622/00_real21_grid.png`、`02_night_discs_9presets.png`）；回归无崩溃。
  与 `stf_apodization`（纯强变迹 2.5、无球差）区别：cream 有球差亮心 + 中等变迹。
- **影响**：soap 保持 D47 强球差（过矫正的亮环是"边缘能量堆积"、非"保留结构"，用户未反馈 soap 没虚化）。
  apodization 是 ctrl_vec 外扩展项（D36，不进训练采样）；cream 预设用于 demo/showcase/指纹。
- **教训**：「奶油柔散」的【柔虚化】应主要靠【变迹（振幅）】而非【欠矫正球差（相位/亮心）】——后者给的是
  "亮心 + 保留焦外结构"（Helios 式），与"均匀柔虚化"是两种不同视觉。建模时须区分 PSF 的"亮心"vs"柔边填充"。

## D47 — soap/cream 忠实复刻真实镜头：强球差 W040+W060（用户选"忠实复刻"取向）

- **决策**：`soap_bubble` 预设 `W040 1.5→2.0 + 新增 W060=0.8`；`cream_soft` `W040 −1.5→−2.0 + W060=−0.8`。
- **背景**：用户反馈 soap/cream 观感"奇怪"。联网对照真实标准（phillipreeve / 4photos Trioplan 评测）：
  **soap(过矫正)=锐利明亮【边缘环】+ 较暗但【填充】的中心 + double-edge；cream(欠矫正)=明亮【中心】
  向边缘【柔和渐隐】、无硬边、低对比**。我们旧 soap(W040=1.5)是【碗状弱环】（中心凹陷渐变、边/心仅1.6）、
  cream(W040=−1.5)【几乎平】（中心/边≈1）——都不像真实。在"强特征↔焦内锐"的物理 trade-off 中
  用户明确选**"忠实复刻"**（接受焦内 glow）。
- **根因 + W060 的关键作用**：① 中等球差 + gamma=2.2 显示 + 大离焦稀释 → 特征弱。② **仅靠 W040 增大
  无法得到真实形态**（W040 大只是碗更深、盘更大）；**高阶球差 W060 才能把 soap 从"碗状"塑成"平底中心 +
  陡峭锐亮环"（边/心 1.6→7.9）、把 cream 塑成"圆顶亮心渐隐"**——这正是真实肥皂泡/奶油的剖面。
- **trade-off（物理，已向用户说明）**：强亮环/亮心 ⟺ 焦内 glow（强球差既造焦外特征、又造焦内残余弥散圆；
  balance c=1.3 已让焦内尽量好但救不了强球差）。真实 Trioplan/dreamy 镜头焦内同样软（"ethereal glow"）。
  用户选"忠实复刻"=接受这个焦内 glow 作为镜头真实标志。
- **验证**：night panel soap=明亮锐环+暗心填充、cream=亮心柔渐隐（`review_0622/02_night_discs_9presets.png`）；
  real21 灯串=肥皂泡盘 + 枯枝二线性、cream 亮心（`01_real21_bokeh_zoom.png`）；全预设回归无崩溃。
  nisen（W040<0+W060>0 反号）仍区别于 cream（同号）——盘内带状双线 vs 圆顶亮心。
- **影响/可逆**：W060 是扩展像差（D36，`sample_random(extended=False)` 默认不进训练采样、保持 13 维
  ctrl_vec）；soap/cream 预设用于 demo/showcase/指纹反演，训练走 sample_random 不受影响。指纹反演：
  主体 glow 是镜头特性、a 含 W040+W060 可反演。预设系数可调回。
- **教训**：定位"形态奇怪"必须联网对照真实镜头【剖面】（锐环+暗心 vs 碗状）；塑造真实球差盘形态
  **W060（高阶球差）是必要项**，仅调 W040 改不出"平底+锐环"。

## D46 — 焦内锐度修复（balance 用最小弥散圆系数 1.3）+ 撤回 soap W040=2.0 + K 回调

- **决策**：① `_layer_psf` 的球差焦移补偿系数 `1.0→1.3`（`W020_eff = W020_layer − 1.3·W040`）；
  ② `soap_bubble` 预设 `W040 2.0→1.5`（撤回 D45）；③ `auto_focus` 最大 CoC `120→100px`、背景目标 `30→26px`。
- **背景（用户反馈）**：① soap/cream 焦内锐度降低（soap 尤甚）；② 树上红球被"当点光源"渲染、焦外不自然；
  ③ 人群聚集处焦外盘太多、不自然；用户疑"是不是把太多东西当点光源"。
- **澄清（②③不是检测 bug）**：D45 起 `highlight_gain=0`，管线里**已无任何点光源检测/放大**。红球/人群成盘
  是【纯虚化】——任何焦外亮物体虚化都成弥散盘（散景的本质），soap 的亮边让每个盘更显眼、K 过大让盘更大，
  叠加杂乱背景显得"盘太多"。非误检测，是 soap 预设 + 过大 K 在杂乱背景的固有观感。
- **根因（① 焦内软）**：球差镜头"最佳对焦"有两个候选——最小波前 RMS（`W020=−W040`）vs **最小弥散圆直径**
  （`W020≈−1.3·W040`）。旧 balance 用系数 1.0（最小 RMS），但**摄影看的是弥散圆大小（主体清晰度）**，
  欠补偿了球差焦移 → 焦内残余盘偏大。实测焦内 90% 能量半径：W040=2.0 时 **5.8px(c=1.0)→3.7px(c=1.3)**、
  W040=1.5 时 4.3→2.9px（锐化 ~35%）；且焦外大离焦盘的亮边几乎不变（偏移量相对大离焦很小）。叠加 D45 误
  把 soap W040 增到 2.0（焦内更软），故②撤回到 1.5。
- **验证**：real21 焦内女生脸/头发锐度恢复到接近 ideal（`_diag_balance13`）；soap 整体=主体清晰 + 背景
  虚化适中（K 132 vs 旧 158）+ 红球/人群盘更温和自然（`review_0622/00_real21_grid.png`）。
- **影响/可逆**：balance 系数 1.3 对所有球差预设统一生效（焦内更锐、焦外不变），物理更正确（最小弥散圆
  是摄影对焦准则）；改动在 `_layer_psf`、合成训练 GT 同步受益（待重训）。soap W040/K 均向更保守回调。
- **教训**：上轮（D45）为追"亮边更明显"增大 W040，忽略了它对焦内锐度的代价；应优先用【物理更正确的
  对焦准则（最小弥散圆）】解耦"焦内锐度 vs 焦外特征"，而非靠加大像差系数（同时恶化焦内）。

## D45 — gamma 回归标准 2.2（修正 D44 的对比压平）+ 背景虚化不足修复 + soap 二线性增强

- **决策**：① 撤回 D44 的 `gamma=4`，demo/e2e/lights_panel 回归**标准 sRGB gamma=2.2**（RenderControl 默认）；
  ② `auto_focus_params` 的最大 CoC 上限 `60→120px`、背景 CoC 目标 `22→30px`；③ `soap_bubble` 预设
  `W040_spherical 1.5→2.0`（亮边/二线性增强）。
- **背景（用户反馈 4 问题）**：① 渲染对像素响应太强、虚化有颗粒感、不柔和；② soap 后景树枝二线性不明显、
  要亮边更窄更明显；③ cream 点光源像没化开；④ 背景虚化量相对前景显然太少。
- **根因**：
  - **①②③ = D44 的 gamma=4 压平了球差盘内对比**。实测 geom soap 盘亮边在【线性域】是 6.5×，但
    `linear_to_srgb` 的 `**(1/γ)` 显示编码把它压成 `6.5^(1/γ)`：γ=4 → **1.6×（亮边几乎消失）**，
    γ=2.2 → 2.3×（保留）。所以 D44 为提亮散景球而用的高 γ，副作用是【幂律压平所有盘内对比】→ soap
    无亮边（②）、cream 无亮心（③）；且高 γ 把每个焦外亮像素都提亮 → "像素响应太强/颗粒"（①）。
    **gamma 是幂律、压对比；乘性提亮才保对比**。标准 2.2 是物理正确显示编码。
  - **④ = auto_focus 的"最大 CoC 60px"上限误伤背景**。该上限为【旧 wave 模式 FFT 采样上限 W020<N/8】
    设；当画面有极近前景（贴镜松枝，视差≈0.8）时 max_dev 大 → 上限把 K 压到 79 → 背景 CoC 仅 10px
    （联网查证：CoC 对仿射视差线性、公式本身对，问题在 K 标定的上限）。**geom 模式无采样上限**（D40）→
    放宽到 120px → K=158、背景 CoC 20px、散景球明显（物理上极近前景本就该糊、不该为压前景牺牲背景）。
- **验证**：night panel soap 亮边环/cream 亮心恢复（vs gamma=4 平盘）；real21 整体更柔和、背景虚化
  10→20px、soap 枯枝二线性双线明显（`review_0622/01_real21_bokeh_zoom.png`、`02_night_discs_9presets.png`）。
- **与 D44 的关系（诚实记录迭代）**：D44 的"gain 激进放大从 LDR 猜测→溢出/误判"诊断**仍成立**；但 D44 用
  `gamma=4` 补偿散景球亮度是【错误的补偿手段】（压球差对比）。D45 修正：用**标准 gamma=2.2 + 接受 LDR
  输入下散景球偏暗**（纯夜景可选开温和乘性 highlight_gain），而非高 gamma。散景球的"亮"本应来自点光源
  真实 HDR——LDR 输入无此信息，不应用非物理 gamma hack 伪造。
- **影响/可逆**：synth 训练数据对齐 gamma=2.2（核心合成层改动，待重训）；soap W040=2.0 仅改预设、训练走
  sample_random 不受影响；K 上限放宽对 wave 模式需注意采样上限（docstring 已注明）。
- **教训**：一个旋钮（gamma）承担两个职责（显示编码 + 亮斑化）时，调它会有隐藏副作用（压对比）。
  职责应分离：显示编码用标准 2.2，亮斑化用乘性/HDR。

## D44 — 高光管线根本检讨：回归 BokehMe 式【gamma 域亮斑化】，撤回激进 HDR 放大默认 + 叶片曲率参数化

- **决策**：① demo/e2e/lights_panel 的高光处理从"激进 HDR 放大"改为 **BokehMe 式 gamma 域亮斑化**
  （渲染 `gamma=2.2→4`、`highlight_gain` 默认 `30→0`）；D38/D41/D42 的放大机制**保留为可选**（默认关，
  文档标注温和量级）。② 新增光圈叶片**边缘曲率** `blade_curvature∈[0,1]`（0=直边正多边形→1=圆，
  角保持尖），`sample_random` 采样、`hexagon` 预设设 0.25。
- **背景（用户反馈，强 regression 信号）**：点光源弥散圆仍溢出、能量被放大好几倍；**原图没有点光源
  的地方（雪地/天空/脸）虚化后也冒出弥散圆**；"还不如三天前自然"；hexagon 边太直、真实光圈叶片应有曲率。
- **根因（对照 BokehMe 论文 + 联网查证）**：D38 把一个**可选的、温和的**高光增强，错误地做成了**默认的、
  激进的**放大。实测 BokehMe 原版：(a) 散景球的"亮"靠 **gamma=4 准线性域** `render(image**γ)**(1/γ)`
  ——暗背景在 `**γ` 域趋近 0 → 散景盘归一化后**保持亮点本身亮度、不被背景稀释、绝不超过原值（不溢出）、
  无模糊区恒等**；(b) 它的"高光增强"（=我们 `highlight_gain` 同源）**只 ×1.4 且 `--highlight` 默认关**。
  而我们 `gain=30` 从 **LDR 8-bit 图猜测放大 30 倍**（BokehMe 训练数据是 Blender EXR 真 HDR、无需猜）：
  ① 真点光源放大 30 倍 → 溢出死白；② **天空(>0.82 占 88.7%)/雪/脸等非点光源亮区被绝对阈值误判** →
  放大 30 倍 → 虚化冒**假弥散圆**。三天前 gain=8 放大温和故"更自然"。
- **理由**：我们没有 HDR 输入，从 LDR 猜测放大几十倍必然误判+溢出；gamma 域亮斑化是**有界、不误判、
  不溢出**的（雪地/天空被"误判"也只是 `**γ` 域恒等变换、虚化后仍是雪地亮度，不会冒刺眼弥散圆）。
  这是 BokehMe 的务实智慧——温和增强下根本不需要精准检测。
- **验证**：real21 灯串散景球从【死白连片】→【柔和暖金、有层次、不溢出】；前景雪地不再冒假弥散圆；
  9 预设形态清晰（`review_0622/00_real21_grid.png`、`02_night_discs_9presets.png`）；blade_curvature
  扫描 0→0.7 边从直线渐变到弧形、角保持尖（`_diag_blade_curvature` 验证），hexagon 圆角弧边自然。
- **影响/可逆**：① synth 训练数据应**对齐 gamma 域（gain 小/0）**再重训——这是核心合成层面改动
  （用户已确认要动核心合成）；② `highlight_gain>0` 仍可选用于【已知强点光源】场景的温和增强
  （配 hdr_thresh/软肩，D42/D41）；③ blade_curvature 仅扩展 AberrationCoeffs，向后兼容（默认 0=旧直边）。
- **教训**：把别家"可选+温和"的设计当成"默认+激进"来用，且在缺 HDR 输入时强行猜测放大，是这轮
  regression 的根。修复方向应优先对照原论文的**默认行为与量级**，而非只看机制名称。

## D43 — 灯珠嵌套环根治（高光核心深度统一）+ swirl 旋焦增强

- **决策**：① `render/renderer.py` 新增 `unify_highlight_core_depth`：把每个【真饱和高光核心】
  (detect>0.92) 连通区的视差统一为该区【最亮像素】（灯珠中心）的深度；demo/e2e 渲染前调用。
  ② PRESETS 的 `swirl_catseye` / `double_gauss_edge` 的 `vignette_strength` 0.5→0.9、
  `vignette_radius` 1.2→1.05（猫眼切得更狠）。
- **背景**：用户反馈 ②ideal 光斑外围套一圈淡盘（嵌套同心圆）③soap 光斑中间空/外套淡圆
  ④swirl 旋焦不明显。
- **理由（②③嵌套环）**：逐层排查证明【不是 PSF/渲染器 bug】——geom PSF 单盘剖面在所有离焦下
  **边/心=1.00 完全均匀**、孤立单深度灯珠渲染盘外干净、单灯珠+背景各种配置都不产生亮边环。
  真因：真实图里 DA V2 对小过曝灯珠的深度估计不可靠，**同一灯珠的像素被判成不同视差**（实测灯串
  边界局部 std 高达 0.36 → K=79 下 CoC 差 28px）→ tent 分层把灯珠拆到【不同 CoC 的层】→ 大小
  不一的盘叠加 = 嵌套。统一核心深度 → 单灯珠单盘 → 嵌套消除（实测球从"暗心亮边"变"均匀实心"，
  灯串区视差均值前后不变=副作用极小）。**用最亮像素深度而非中位数**：中位数受边缘背景污染、且实测
  恰落焦带把灯珠推成合焦清晰；最亮像素=灯珠核心，深度最具代表性。
- **理由（④swirl）**：猫眼/旋焦强度 ∝ 像高 H × vignette_strength，旧 0.5 在画面边缘仍太弱。
  增强后画面四角散景盘明显被切成【朝画面中心的眼形】、旋焦感清晰（`_diag_swirl_sweep` 验证）。
- **关键澄清（非 bug、不强行消除）**：**密集灯串相邻散景球的加性重叠会形成蜂窝状亮纹**
  （看似每个球有亮边）——这是【物理真实】（真实密集灯串散景同样如此，两球叠加确实更亮），
  不是缺陷。D42 降 gain + 只放大真饱和核心已显著减轻（成盘高光变少→重叠减少）；完全消除需场景
  灯不那么密集，不应在渲染器层面强行抹平。
- **影响/可逆**：`unify_highlight_core_depth` 不可微（用 scipy），但**深度是网络输入、不在像差系数
  梯度路径上** → 不破坏指纹标定；仅用于 demo/e2e 的【真实图】（合成训练 GT 深度精确、不需要）。
  swirl 增强仅动 PRESETS（demo/showcase 用），训练走 `sample_random` 不受影响。

## D42 — 散景球过曝死白根治：WR 的【HDR 放大阈值】与【高光检测阈值】分离

- **决策**：`RenderControl` 新增 `highlight_hdr_thresh`（默认 0.92），WR 的逆色调映射 `sat` 改用它
  （而非检测阈值 `highlight_thresh`≈0.82）算：`sat=clip((d−hdr_thresh)/(1−hdr_thresh),0,1)`、
  `peak=1+gain·sat`。检测掩膜 `m` 仍用 `highlight_thresh`。demo/e2e 默认 `gain` 60→30。
- **背景**：用户反馈"灯串点光源虚化后过曝、变高亮死白圆盘，很不自然——**原图灯珠甚至没过曝，
  为什么输出却放大了能量？**"。实测：原图灯串核心**确实饱和到 1.0**（0.89% 像素 >0.98），但 WR
  把能量**放大 59 倍**（gain=60）；根因是 `sat` 从检测阈值 0.82 起就线性放大 → **d=0.9 的中等亮度
  灯珠也被放大到 peak=27**（凭空造能量），密集散景球重叠后死白。
- **理由**：物理上**只有被传感器裁剪到接近饱和 (d→1) 的点才丢失了 HDR 信息、才该恢复放大**；
  d<hdr_thresh 的中等灯珠并未过曝、真实亮度即原值，放大它就是凭空造能量。分离阈值后：中等灯珠
  `sat≈0、peak≈1`（照样做方波散景盘、但不放大 → 保留颜色不死白）；真饱和核心 `sat→1、peak→1+gain`
  （成明亮散景球）。正面回答用户质疑——未过曝的灯珠本就不该被放大。
- **验证**：21.jpg 灯串区死白占比 **26.9%→11.3%**（hdr_t=0.95,gain=30）、彩色度 0.271→0.333；
  精确算术：d=0.9 灯珠 **peak 27→1**、d=1.0 真饱和核心保持 1+gain；视觉上散景球由死白变【柔和暖金
  色、有层次】（`outputs/samples/review_0622/01_real21_bokeh_zoom.png`）。
- **影响/可逆**：改变了有高光样本的渲染分布 → 生产细化网重训应连同 geom(D40)/软肩(D41)/本条一起；
  `highlight_hdr_thresh=highlight_thresh` 即恢复旧行为。残留死白来自密集球加性重叠（见 D43，物理真实）。

## D41 — 散景盘重叠亮度溢出根治：末端【HDR 软肩色调映射】（替代硬裁）

- **决策**：在 `render/renderer.py:linear_to_srgb` 的 gamma 编码【之前】插入软肩色调映射
  `_soft_shoulder(L, knee)`：`L≤knee` 恒等、`L>knee` 指数滚降 `knee+(1−knee)(1−e^{−(L−knee)/(1−knee)})`
  渐近逼近 1（永不超过、C1 连续可微）。新增 `RenderControl.highlight_tonemap=True`（默认开）、
  `tonemap_knee=0.8`。仅在 `highlight_gain>0` 时启用（无 HDR 高光时内容本就 ≤1、跳过保持 byte 级不变）。
- **背景**：用户反馈"焦外虚化盘叠在一起时亮度溢出"。实测密集灯串（100 点）下 **40% 像素线性能量 >1.0**
  （最高 1.3~1.4）。根因链：① WR 把过曝点源提到 HDR 平顶 `1+gain`（如 61）；② 同层散景盘是**加性光**
  （`out=conv(lin,psf)`，黑底保持黑、灯点扩成盘并在重叠处**相加**）——这是**物理正确**的；③ 旧
  `linear_to_srgb` 只 `clamp(min=0)`、**不处理上界** → 超 1 的加性结果在保存为 8-bit 时被**硬裁成平白块**，
  同一片纯白、丢失盘边界与红蓝叠加层次。
- **理由（联网查证，不局限本地论文）**：散景盘重叠该不该相加？**该**——两个独立光源在传感器处辐照度
  相加，真实相机重叠高光确实更亮且可饱和到白。所以**不能改成 max/screen**（非物理）。标准做法
  （MJP《How To Fake Bokeh》、Bart Wronski DoF 系列、多篇 DoF 专利）一致：**HDR 加性合成 + 末端色调
  曲线滚降**，而非硬裁。软肩让 1.0~1.4 的重叠区映射到 0.93~0.99 而非全 1：重叠核间仍有梯度→盘边/
  层次可见，极亮处仍逼近白（真实饱和高光本就近白）。色调映射放 gamma 之前 = 标准 scene-referred 管线
  （HDR→tonemap→显示域→OETF）。
- **为何用 knee 软肩而非全局 Reinhard**：全局 `L/(1+L)` 会把中间调一并压暗（0.5→0.33）、改变整张照片
  曝光。knee 软肩**对 `L≤0.8` 完全恒等**（主体/正常曝光不动），只压高光溢出，是"保 LDR、只救 HDR 溢出"
  的最小干预。
- **验证**：密集灯串 **溢出 40.1%→0%**、最高 1.307→0.999；近白 [0.9,1] 区间占比 18.4%→58.5%（被裁的平
  白区救成有梯度的近白渐变）；gain=0 时软肩开/关 **max|Δ|=0（byte 级不变）**；软肩曲线单调、knee 下恒等
  误差 0、上确界=1.0；软肩本身对输入可微（梯度 1.0/0.37/0.05/→0 渐近）。对比图
  `outputs/samples/overlap_tonemap_fix.png`（旧版中心平白块 vs 新版盘层次穿透）。
- **影响/注意**：① 输入与 GT 都走 `linear_to_srgb` → **二者一致地 tone map**（synth GT 同样受益、训练分布
  一致）；② 因改变了有高光样本的渲染分布，**生产细化网重训应连同 geom（D40）一起在新管线上做**；
  ③ **已知正交问题（非本次引入）**：`linear_to_srgb` 的 `x**(1/2.2)` 在 x=0（黑背景）导数→∞、WR 的
  `chroma=lin/detect` 除法，使**对系数回传出现 NaN**（geom/wave、tonemap 开关下**完全一致**，证明与本次
  无关）——指纹标定本就应用掩膜损失、不回传黑背景，D40 也已建议指纹用 wave；留作后续 eps-clamp 加固。
- **可逆**：`highlight_tonemap=False` 即恢复旧硬裁行为。

## D40 — 散景盘软边根治：新增【几何光线散射 PSF】模式（默认），波动光学转可选

- **决策**：在 `optics/psf.py` 新增 `geometric_psf` / `rgb_geometric_psf`，`RenderControl.psf_mode`
  默认 `'geom'`（几何光线散射），`'wave'`（原 |FFT{P}|² 波动光学）转可选。几何模式：每个光瞳点
  的光线落在像面 ∝ 横向光线像差 = `GEOM_SCALE·∇W`（波前梯度），按光瞳强度 A² 双线性 splat。
- **背景（连查三轮才坐实）**：用户先后指出散景盘"硬边发软发厚"。逐项排查证明 **波动 PSF 的盘外缘
  恒为盘半径的 ~13%（软）**——且与盘大小、apodization、光谱平均、光瞳分辨率 N(256→512) **都无关**
  （都只动 8→7px）。根因：我们受 **采样上限 W020<N/8** 约束只能仿真【小离焦】，停在 **衍射 regime**；
  而真实散景是【大离焦】**几何 regime**（硬边）。BokehMe(scatter 硬盘核)、Wu2010(光线追踪) 都靠
  **几何光学** 得硬边——波动光学是我们独有的软边来源。
- **理由**：横向光线像差 = ∇W 是标准几何光学；对【我们自己的单波前多项式】做光线散射，**无需镜头
  处方**（区别于 Wu 的全镜组光追）、**可微**（梯度对 coeffs 流通，支撑训练+指纹）、且：① 硬边
  （落点边界=光瞳边界）；② 焦散自然涌现——soap 亮边环(过矫正)/cream 亮心(欠矫正)/coma 彗尾/
  astig 椭圆/猫眼/多边形/W060 全部正确；③ **无 FFT 采样上限**（可渲任意大盘、无混叠）；④ 无需
  光谱平均/apodization 抹环（几何无衍射环）；⑤ **更快**（3 通道 splat vs 9 次 FFT）。
- **验证**：全渲染器实测 **几何边宽 3~4% vs 波动 17~20%**（锐 4~5 倍）；soap=【填充亮边盘】非空心
  donut；渲染 0.02s vs 0.11s（更快）；可微 dLoss/dW040 有限非零；夜景/21.jpg 全跑通
  （`outputs/samples/geom_vs_wave.png`、`night/soap_bubble.png`、`21_geom_soap.png`）。标定
  `GEOM_SCALE=2.02` 使盘半径=3.93·W020 与波动/calibrate 一致（切模式不改 CoC）。
- **影响/注意**：① **合成 GT 改用 geom**（硬边更真实）→ 生产细化网应在 geom GT 上重训
  （train_run7 是 wave、in-memory 不受影响；其 NAF/film_res 架构结论 PSF-无关、仍有效）；
  ② **指纹标定**：geom 的 splat 梯度较弱（floor 分桶只过亚像素权重）→ 精细反演建议用 `'wave'`
  或后续加软 splat；③ balance_spherical 焦移补偿照常生效（_layer_psf 在分派前已注入）。
- **可逆**：`psf_mode='wave'` 即恢复波动光学（星芒/衍射/小离焦近焦区仍用它）。

- **决策（含一次自我修正）**：① `auto_focus_params` 的 tol 由 `(p15~p85)/2、上限0.15` 收紧为

  改默认 γ=1（关闭）**，函数保留为可选工具（仅手动用于确证被压扁的场景）。
- **关键诊断（先错后纠）**：初判"DA V2 远景压缩致女生(0.18)离背景(0.0)太近"，加 γ 去压缩 → 看似背景更虚，
  但**把 d_f 推高、虚掉了主体的脸**（用户实测女生糊）。深挖发现：我一直在用**低质量缓存 `vitb@518`**
  测试，它把**女生脸与背景树 merge 到同一视差(0.07≈0.07)** → 任何对焦都"脸虚则树虚、树清则脸清"，

## D38 — 用户反馈（P1）：夜景散景盘暗 + 硬边发软发厚 → WR 逆色调映射（饱和 ramp）

- **决策**：把 WR 平顶高度由【固定 `1+gain`】改为 **BokehMe++ §III-C 逆色调映射/highlight enhancement**：
  按像素饱和度 `sat=clip((d−thresh)/(1−thresh),0,1)` 线性抬到 `peak=1+gain·sat`（完全饱和 d=1 → 顶到
  `1+gain`）。`_per_region_peak` 改为【逐高光区取 max(peak) 均匀化】（方波平顶，替换原 bloom 估计）。
  demo/e2e 的 `highlight_gain` 由 8 提到 60；夜景 showcase 用 ~120。
  **盘形与亮度两个独立旋钮（关键，曾踩坑）**：`W040` 控**环/心比(donut度)**、`gain` 控**亮度**。
  曾用 `W040=3 + gain=500` → 环过强成空心环 + 环饱和炸白 = **像折返镜头(catadioptric donut)而非
  soap**（用户指出）。修正：**W040≈1.5（preset 原值，填充式亮边、非空心）+ gain≈120（亮但不炸、
  结构可见、环/心≈1.3）**。
- **背景**：实测旧法点光摊到大盘后盘亮仅 0.16（gain9）、0.50（gain120）——**盘永远到不了饱和**，灰暗软边。
  根因：盘亮 ≈ 源能量×peak/盘面积，小点摊大盘要饱和需 peak~数百，而固定 `1+gain` 的 gain=8~10 太低。
  曾误用 `d/(1−d.clamp(.995))` 逆映射，被 clamp 把 itm 封顶 ~167 → gain>166 失效（盘卡在 0.57，已修正）。
- **理由**：真实路灯亮度是传感器饱和的几百倍，散景球本就死白；逆色调映射从被裁剪的 LDR **恢复 HDR**，
  gain 作"最亮源相对饱和的倍数"上限。盘（或球差盘边亮环）顶到 ≥1 而**饱和**后，边缘从 1.0 陡跌到 0 →
  **视觉硬边**——硬边的根本来自饱和，apodization/光谱只改 8→6px 边过渡（非主因，已验证）。
  另：散景盘**外缘过渡 ~7px 是离焦 PSF 的物理衍射边**（实测 apod/光谱/光瞳分辨率 N=256→512 都只
  动 8→7px，**非 bug、非软化来源**），盘越大该 7px 占比越小越显锐；不要再去"修边软"。
- **影响**：夜景 soap 盘明亮填充、亮边自然（非空心 donut）、ideal 盘平亮、cream 亮心；不炸白。
  合成训练点光本就 d≈1 → peak=1+gain 不变，**训练分布不受影响**。
- **可逆**：`highlight_gain=0` 关闭 WR；ramp 是单行公式。

- **问题发现**：train_run5（Plan B v0，50k）完成后跑 e2e 21.jpg，**主体人脸被明显虚化**——

  ```python
  scale_px = max(K / 8, 2)                  # 过渡宽度随光圈缩放，K=84 → 10.5px
  α_gt = clamp(1 + clamp(r, max=0) / scale_px, 0, 1)
  ```

  合焦像素永远 α_gt=1。

## D35 — 遮挡漏光的【定向修复】：焦带吸附（非全套 Dr.Bokeh 重做）

- **日期**：2026-06-21
- **背景**：D34 诊断出 tent 拆层遮挡漏光（锐前景中心渗背景色 0.35）。讨论是否重做遮挡。
- **关键澄清（纠正 D34 的"不重要=随便修没代价"措辞）**：**简单加性覆盖修法有真实坏处**——
  它会让【虚化叠虚化】过度遮挡：两个不同深度的失焦散景盘重叠时近的会**吃掉**远的，
  破坏散景盘的半透明叠加（夜景灯点展示的核心）。正确全套修法（区分相关/独立用加性/乘性）

- **决策：焦带吸附（`layer_weights` 加 focus_disparity/tolerance 参数）**。漏光只来自

  - **GT 路径（composite_blurred_layers）不用 layer_weights → 训练 GT 不变，仅 B_phys 输入更干净**。
  - 仅 focus_disparity 给定且 tol>0 时激活；3 个 render_* 调用点透传 ctrl 的 d_f/tol。
- **验证（CPU 全过）**：① 锐前景中心 (0,0.35→0,0)0.9,0 纯绿无渗 ✓；② 散景盘重叠区红蓝都>0
  半透明叠加未被吃 ✓；③ 能量守恒 0.5→0.5 ✓；④ 9 预设无崩、无焦带接缝（`d35_occlusion.png`）。
- **范围（诚实）**：本修法只解【锐前景 tent 拆层漏光】。**不含** MPIB 式被遮背景 inpainting
  （虚前景揭示背景时的内容补全）、也不改虚叠虚的乘性模型（独立失焦层乘性本就正确）。
  那些是正交的、更大的 future work，优先级低（细化网兜底 + 非核心贡献）。
- **对 train_run7 影响**：无（在跑进程用旧码）；本修复对【下次】训练的 B_phys 输入生效
  （更干净 → 细化网要修的边界更少，可能缓解 Plan B matte 塌缩）。conditioning.py 的
  layer_weights 未传 focus 参数（焦内描述子≈sharp 不论是否吸附，差异可忽略）。
- **可逆性**：高——layer_weights 新参数默认关（None/0 = 原 tent 行为）；tol=0 即禁用。

- **与他法对比**：几何/生产管线（游戏/shader）用硬几何盘 → 任何离焦都锐边，但**无近焦
  衍射物理、无像差结构**；我们的波动光学是物理正确版，小盘软边是其【优点】不是缺陷。

### 二、新增像差（用户指定补齐，对照真实镜头清单）
- **变迹 apodization（STF/APD 镜头）**：`AberrationCoeffs.apodization`，光瞳振幅乘
  `exp(−a·ρ²)`（`pupil.amplitude`）→ 径向透过率渐暗 → 散景盘**边缘柔化、无硬边**

  预设 `stf_apodization` / `spherochromatic`。验证图 `outputs/render_test/new_aberrations.png`。
- **像差总数 9→11**：波前 5 + 振幅 3（多边形/猫眼/**变迹**）+ 色差 3（LoCA/LaCA/**球色差**）。
  仍缺：高阶（5 阶+）球差（精确复刻具体镜头的硬伤，future work）、洋葱圈（D15 故意移除）。
- **可逆性**：高——新字段默认 0（无效应），不影响既有预设/训练分布；apodization 可微（exp）。

## D32 — P2a：NAFNet 块替换 FiLMResBlock（细化网骨干升级）

- **日期**：2026-06-19
- **背景**：v1 的 `FiLMResBlock`（plain conv 3×3 ×2 + FiLM）参数效率已被 NAFNet 系
  明显超越（NETWORK_DESIGN §5.1；NAFBET 已在散景变换任务验证）。在 train_run6
  (Plan B + D30 + AMP) 跑通后，按用户指定推进 P2a。
- **决策**：实现 `NAFFiLMBlock` 作为 `FiLMResBlock` 的【可切换】替代（`block_type`
  开关，v1 仍可复现作消融基线）。结构：
  - 空间子块：LN → **FiLM(挂在 LN 之后)** → 1×1↑2C → 3×3 深度可分 → **SimpleGate**(乘性门, 2C→C)
    → **SCA**(简化通道注意力) → 1×1→C，残差缩放 β_s。
  - 通道子块(FFN)：LN → 1×1↑2C → SimpleGate → 1×1→C，残差缩放 γ_s。
  - SimpleGate（通道对半相乘）替代激活函数 + 深度可分卷积，是 NAFNet 参数效率的核心。
- **关键发现：NAF 块在同 ch 下约【半参】**（matte 网同 96/48 ch：film_res 706,917 →
  naf 364,005）。为做 §5.1 "**同参数换块**" 的公平对照，把 naf 加宽到
  **ar_mid=144/iu_mid=72 → 712,821**（≈ film_res 同预算）。两种对照都有价值：
  - naf@96/48（半参）：测"能否半参不掉点"（RQ2 轻量故事，更强）；
  - naf@144/72（同参）：测"同预算下块是否更好"（§5.1 本意，决定是否采用 naf）。
- **残差缩放 β_s/γ_s 初始化=1（标准残差）**，不取 NAFNet 的 0 初始化：0 初始化会让块
  首步梯度只到缩放系数、暂时阻断卷积/FiLM；本项目靠近零的 head 已保证输出稳定起点
  （见 ARNet.head 注释），故无需再靠 0 残差缩放求稳，1 初始化让块首步即正常训练。
- **落地**：`network.py` 加 `LayerNorm2d`/`NAFFiLMBlock`/`_make_block` 工厂 + ARNet/IUNet/
  RefineNet 的 `block_type` 透传；`train.py` 加 `--block/--ar-mid/--iu-mid` 与 config
  `block_type/ar_mid/iu_mid`，checkpoint 存这三键；`e2e_demo.py` 按 ckpt 重建网络。
  smoke 验证：前向/反向通、梯度达 net+ctrl_vec、参数量三档（707k/364k/713k）符合预期，
  CPU 训练 3 步 loss 正常。
- **首个 P2a 实验**：naf@144/72（同预算），50k，AMP+batch6，COCO，train_run6 完成后接力启动。
  验收：合成边界 L1 ≤ film_res 基线（§5.1 预期 0.3–0.5dB 级改善）。
- **可逆性**：高——纯新增可切换块，`block_type=film_res` 完全复现 v1；旧 ckpt 无新键自动回落。

## D31 — 弥散圆"中心亮点"伪影：分层数不足 → 自适应 n_layers 限制 CoC 步长

- **日期**：2026-06-19
- **问题发现**：肉眼发现散景盘（弥散圆）中心有一个不自然的亮点（夜景灯点展示
  `demo_lights_grid` 最明显）。诊断排除了 PSF 本身（孤立点源的 PSF 中心/盘比 ≈1.0，
  正常）——伪影由【渲染器分层】引入。
- **根因（本质 = D9 "双环鬼影" 的层合成版）**：`render_*` 从估计视差出发，用
  `layer_weights` 的 tent 软权重把每个像素分给相邻两层。层中心固定在视差 [0,1] 上
  均匀排布，相邻层 CoC 步长 = K/(n−1)。**当 K 大、层少时步长巨大**（K=84,n=8 → 12px/层）：
  一个位于中间视差的点光源被 tent 拆成"65% 给 CoC=17px 的层 + 35% 给 CoC=5px 的层"，
  合成后 = 大盘上叠一个小亮盘 = **中心亮点**。实测中心/盘比：n=8→1.91，n=12→1.06，
  n≥16→≈1.0。**注意 GT 路径无此问题**：`composite_blurred_layers`（合成训练 GT）按
  每个图层的【精确视差】各算一次 PSF，从不用 tent 分层 → GT 干净；伪影只在
  "估计视差→tent 分层"的 B_phys 路径，且因网络只修边界、盘心在平坦区(band 外)
  会原样透传到 e2e 输出。
- **决策：自适应分层数** `RenderControl.effective_n_layers()`：
  ```
  n_eff = clamp( max(配置 n_layers, ceil(1 + K/s_max)), 配置 n_layers, 40 )
          s_max = MAX_COC_STEP_PX = 3.0px（实测 ≤3px 时中心/盘 ≤1.0）
  ```
  - **只增不减**（下限=配置 24）：K≤69 完全不变（无回归、不动既有训练分布），
    仅大虚化(高 K)加层（K=84→29, K=96→33, K=120→40 封顶）。
  - 空层被渲染循环跳过 → 景深集中时几乎不增算力；封顶 40 防显存/算力失控。
  - 分层数是离散选择、不可微 → 对 K 取 detach 的 float，不影响指纹标定梯度。
- **落地**：`renderer.py` 三个 `render_*` 与 `conditioning.py`（描述子表 + tent 条件图）
  全部改用 `ctrl.effective_n_layers()`，保证条件与渲染口径严格一致（smoke 验证层数匹配）。
  另修 `demo.py` 夜景灯点面板硬编码的 `n_layers=8`（移除→走自适应 24）。
- **验证**：K=40 单点源中心/盘 1.91→0.96；K=84 可见盘(r≥8px) ≤1.00；`demo_lights_grid`
  重渲后盘面均匀（cream 亮心/soap 亮环/ideal 平 各自正确，无杂散中心点）。
- **对运行中的 train_run6 的影响**：无（Python 模块已加载，文件编辑不影响在跑进程）；
  本修复对【下一次】训练生效。train_run6 的 B_phys 用旧 24 层（synth K 范围内步长 ≤4px，
  基本干净），与新代码 e2e 评测(29 层)的物理差异极小、网络只修边界，可接受。
- **可逆性**：高——单个方法 + 调用点替换；常量 `MAX_COC_STEP_PX`/`N_LAYERS_CAP` 可调。

## D8 — M2 渲染器路线：**分层 gather + tile 近似**，不从零复刻 Dr.Bokeh
- **日期**：2026-06-10
- **背景**：项目自审发现 M2 是最大可行性风险：D6 的"渲染时按像素方位角旋转 PSF"
  与任何高效卷积/scatter 实现都冲突——逐像素旋转 PSF 又慢又难写难调；
  且 Dr.Bokeh 的精确遮挡机制对初阶开发者从零复刻难度过高。
- **决策**：渲染器采用**分层 gather + tile 近似**：
  ① 按视差分 K 层；② 图像切 tile（如 64px），每个 tile 用其中心 (像高 H, 方位角)
  对应的 PSF（从 +x 约定的 PSF 旋转一次到该 tile 朝向）做 FFT 卷积；
  ③ 层间前后 alpha 合成 + 归一化处理遮挡。Dr.Bokeh 式精确遮挡留作改进项而非起点。
- **理由**：① 空变模糊的标准做法（refs 里 MPIB 同族），**可微、显存可控**；
  ② 旋转从"逐像素"降为"逐 tile"，D6 的字典省显存收益保住、开销可控；
  ③ 每层每 tile 都能单独可视化，契合"产出可验证"协作规范；
  ④ tile 内 PSF 视为常量的误差 ≪ 视场变化尺度（tile 64px vs 半幅 ~1000px）。
- **影响**：`render/renderer.py` 按此实现；CLAUDE.md 第 3 节增补"渲染器设计约定"。
- **可逆性**：中——接口不变的前提下可日后替换为 Dr.Bokeh 式精确渲染。

## D9 — PSF 字典降级为**推理缓存**；训练/标定按层**现场算 PSF**
- **日期**：2026-06-10

- **理由**：① 彻底绕过插值伪影；② 对 a 的梯度链路更直接（无插值中介）；
  ③ 计算量实测可忽略，不破坏 12GB 约束。
- **影响**：psf.py 模块头与 docstring 已改写定位；M2 渲染器主路径不依赖 sample_psf。
- **可逆性**：高——两条路径并存，按场景选用。

## D10 — PSF 物理修正三件套：LaCA 改径向平移 / 保留口径蚀透过率 / 混叠 guard
- **日期**：2026-06-10
- **背景**：项目自审发现三处物理建模问题。
- **决策与理由**：
  ① **LaCA 从"径向缩放"改为"径向平移"**：横向色差的本质是各通道放大率不同 →
    离轴处各通道 PSF 中心彼此**错开**（平移 ∝ H），不是绕自身中心缩放
    （缩放更接近球色差，渲染不出真实的单侧紫/绿镶边）。实现沿 +x 平移
    `laca_rgb[c]·H` 像素，与 D6 猫眼约定一致、渲染时随方位角一起旋转；
    旧实现还有 bug：缩放未乘 H，画面中心也会被错误施加色差。
  ② **保留口径蚀相对透过率** `pupil.relative_transmission`：PSF 逐个归一化 sum=1
    丢掉了猫眼的"边角失光"分量；T(H)=ΣA²(含猫眼)/ΣA²(无猫眼) 存入字典，

## D13 — 用户审图发现的四个渲染问题：根因与修复（含一个 90° 旋转真 bug）
- **日期**：2026-06-10
- **背景**：用户人工检查 `outputs/render_test/` 指出四个问题。逐一诊断：

  之前用顶边亮斑验证恰好是退化情形，误判为正确。**轴向旋转必须用对角位置验证**。
  修复后点阵图四角猫眼呈切向排列（正确旋焦）。

  高光增益 4→8；另加合成"夜景灯点"展示面板 `demo_lights_grid.png`（盘结构的
  决定性展示，真实照片的灯往往太暗太小）。

  的一侧**（边缘光线折返成焦散环），同号侧为奶油。本项目约定背景(D<d_f)→W020<0，

## D14 — 渲染控制新增**对焦容差 δ0** 与**球差焦移补偿**（解决"主体被虚/虚化过量"）
- **日期**：2026-06-10
- **背景**：用户反馈 demo 虚化量过大且主体也被虚。诊断出两个独立原因：
  ① DA V2 的**相对视差**会把主体自身拉伸出 0.1~0.3 的跨度（相对深度归一化的固有
  现象，非误差），K 稍大主体内部的视差差异就被渲染成可见模糊；
  ② 球差预设（W040≠0）在 W020=0 焦面本就有满幅光晕——但真实摄影中球差伴随
  **焦移**，摄影师会重新对焦到"最佳焦面"，用离焦抵消部分球差。
- **决策**：`RenderControl` 新增两个控制：
  ① **focus_tolerance δ0**（视差半带宽）：r = K·sign(δ)·max(|δ|−δ0, 0)，景深带内
  CoC=0。demo 中 δ0 取主体区域视差 p15~p85 半宽（实测 ≈0.10）。
  ② **balance_spherical_focus**（默认开）：所有层叠加恒定 W020 偏移 −W040
  （波前平衡：ρ⁴−ρ² 是平衡球差，W020=−W040 时 RMS 最小，等价于"对焦到最佳焦面"）。
  另把 demo 的虚化目标调温和（背景有效 CoC≈22px 目标、全图上限 60px）。
- **结果**：soap_bubble/cream_soft 的主体核心收紧（眼镜/纹理清晰，残留轻微柔光
  =球差镜头实拍观感），背景肥皂泡/奶油形态保留；指纹自标定回归不受影响
  （0.8%/0.0%/0.2%）。
- **影响**：两个控制都在 c 的可微链路上（relu 次梯度 / −W040 线性项）；
  训练（synth 默认 δ0=0）与指纹标定的前向模型保持一致。
  开放问题：synth 是否随机采样 δ0 并纳入 ctrl_vec（现 14 维），接细化网时定。
- **可逆性**：高（均为可选项，默认行为有 flag 控制）。

## D18 — 边界细化网设计：残差式 ARNet+IUNet 轻量版 + FiLM 条件 + 误差图融合
- **日期**：2026-06-11
- **背景**：M2 后半接入唯一可训练模块（refine/network.py）。BokehMe 的 ARNet 是
  "从 (图, 离焦图) 从零神经渲染散景"，本项目的物理渲染器已把像差风格做对，
  网络职责收窄为"修深度边界瑕疵"——结构据此改造。
- **① 残差式而非从零渲染**：ARNet 预测的是相对 B_phys 的残差，IUNet 预测的是
  相对上采样粗结果的残差；最终按误差图融合 B = m·B_neural + (1−m)·B_phys。
  理由：物理层的像差风格（肥皂泡/猫眼/色差）不该被网络重绘；小容量网络
  (684k 参数) 学局部修复远比学全图渲染可靠，可控性也不会被网络"洗掉"。
- **② FiLM 条件注入**：13 维 ctrl_vec → 2 层 MLP 共享编码 → 每个残差块一个
  线性头出逐通道 (γ,β)，h←h·(1+γ)+β。渲染器与细化网共享同一个 c
  （CLAUDE.md 第 2 节的一致性要求）。另把 r̃=K·(D̃−d_f)/128 作为输入通道
  （"离焦提示"），物理一致、零成本——依赖 ctrl_vec[0]/[1] 索引，synth 与
  network 两处注释互指。
- **③ 近零初始化，不能严格置零（踩坑）**：输出头若 W=0，d(head)/d(h)=W=0 会把
  流向 FiLM/cond 的梯度整体阻断（冒烟测试实测 ctrl_vec 梯度=0），条件分支
  永远学不动。改为 std=1e-4 正态初始化：起点仍≈透传物理渲染
  （|B−B_phys|=4e-4），梯度全通。
- **④ 训练输入侧与推理严格同路径**：扰动视差 → snap_disparity_edges →
  render_field_patch（单 PSF 视场近似，与 GT 的 composite_blurred_layers
  共享 (H_field, azimuth)，且同样乘 T(H) 保持亮度口径一致）。渲染在 no_grad
  下进行——训练时 c 是给定值，不需要梯度过渲染器（指纹标定才需要）。
- **规模**：ARNet 96ch×3 块 @1/4 分辨率 + IUNet 48ch×2 块 @全分辨率，
  共 684,035 参数（上限 2M 的 34%）；512²/batch4 实测 0.45s/it、峰值 5.7GB。
- **可逆性**：通道数/块数是构造参数，容量不够时可上探（2M 内余量 3 倍）。

## D19 — 误差图融合的"塌缩"问题：三项损失（显式 mask 监督 + 神经分支直监 + 边界加权）
- **日期**：2026-06-11
- **现象（首次 1500 步训练实测）**：只用融合输出的 L1+LPIPS 训练时，误差图 m
  全图→0，网络退化成纯透传（L1(B)≡L1(B_phys) 至小数点后 4 位，最终可视化
  中 |B−GT| 与 |B_phys−GT| 完全相同）。
- **根因（门控的先有鸡还是先有蛋）**：神经分支初始略糊（粗分支经过一次下/上
  采样）→ m>0 处损失更大 → 梯度把 m 压到 0 → m=0 后 ∂L/∂B_neural·m≈0，
  神经分支再也学不动。
- **对策（BokehMe 同款思路，三项加在 train/train.py）**：
  ① **误差图显式监督**：m_gt = clamp(10·|B_phys−GT|, 0, 1)（物理渲染的真实
     误差，phys/GT 都无梯度，是常量目标），L1(m, m_gt)，λm=0.2；
  ② **神经分支直接监督**：L1(B_neural, GT) 不经 mask 门控，λn=0.5——
     保证 m≈0 处神经分支照样有梯度；
  ③ **边界加权重建**：逐像素权重 1+4·m_gt（边界高误差区只占 ~8% 像素，
     不加权时边界梯度被平坦区稀释 ~12 倍），除以权重均值保持量纲。
- **验证**：加 ①② 后重训 1500 步，m 不再塌缩且精确点亮在物体边界
  （outputs/train_test2/viz_it001500.png）；但边界区域 L1 尚无净收益
  （0.0499→0.0498）——1500 步太短 + 梯度稀释，③ 即为此而加。
  长训练（10k 步）的定量结果记入 PROJECT_STATUS §6。
- **新增评测口径**：`evaluate_boundary`（train/train.py）——只统计 m_gt>0.2
  像素的 L1，phys vs fused 对照。全图 L1 被 >90% 平坦区主导，对"修边界"
  这个任务不灵敏，**细化网的净收益一律看边界 L1**。
- **可逆性**：λn/λm/边界权重在 configs/base.yaml train 节，可调可关。

## D20 — 端到端推理：细化网用 512² 滑窗 + 窗心视场条件 + Hann 融合（不整图单次前向）
- **日期**：2026-06-11
- **问题**：细化网在 512² 裁块上训练，且每个样本整块共享一个视场位置
  (H_field, azimuth)（synth 的单 PSF 视场近似）。真实整图推理时这两维怎么给？
  整图单次前向只能喂一个 (H, azimuth)，与"画幅各处视场不同"矛盾。
- **决策**（render/e2e_demo.py `refine_full_image`）：512px 窗口、256px 步长
  滑窗推理；每窗用**窗口中心**的 (H, azimuth) 拼 ctrl_vec（像高/方位角几何与
  render_tiled 完全同约定），各窗输出 Hann² 窗加权融合。
  - 与训练分布最贴近：窗口尺寸=训练裁块尺寸，视场条件=训练的"整块共享"口径；
    与渲染器的 tile 近似（D8）哲学一致。
  - 代价可忽略：0.7M 网络一窗 <10ms，1024 宽整图 ~35 窗共 ~0.2s。
  - Hann 窗边缘值 clamp 1e-3（四角只有单窗覆盖，权重恰为 0 会 0/0）。
- **易错点（实测踩坑）**：synth 的 azimuth ∈ [0,2π)、ctrl_vec 存 azimuth/π ∈
  [0,2)，而 math.atan2 返回 [-π,π]——必须先 `% 2π` 再除 π，否则负角超出训练分布。
- **验证**：512² 合成样本上滑窗 vs 直接前向，边界 L1 0.0353→0.0314（滑窗）
  vs 0.0312（直接），收益保留；|diff|max≈0.011 来自窗心 H 与样本 H_field 不同，符合预期。
- **已知现象（待 50k 正式权重复核）**：train_run1 的 10k 权重（25 张占位背景训练）
  在真实照片上把虚化前景的细枝纹理"找补"回锐利——背景池过小的过拟合，
  这正是换 COCO 118k 背景跑正式训练的动机；机制层面滑窗管线本身已验证无误。
- **可逆性**：patch/stride 是函数参数；若 50k 权重下整图单前向（H=0）质量无差，
  可简化回单前向。

## D21 — 细化网 v2 升级路线：条件机制优先于容量；新增 NETWORK_DESIGN.md 承载完整设计
- **日期**：2026-06-11
- **背景**：用户提出"能否设计更强网络辅助/指导光学瑕疵的呈现"。为此做了一轮

- **首要澄清（红线重申）**："哪里出现何种瑕疵、以何种方式出现"由物理渲染器精确

  ① 散景文献的条件注入停留在黑盒水平（镜头 one-hot/光圈数值嵌入 → f 值控注意力

  ③ NAFNet 骨干在散景任务的有效性已被 NTIRE23 冠军线方案（NAFBET）验证；
  ④ 撞车检查：BokehFlow/Generative Refocusing/BokehDepth（2025.11–12）均未触及
    像差级控制；Voigtländer 2026.02 实体球差控制环镜头反证需求真实性；
  ⑤ 数据集机会：BETD（双镜头转换 GT）可把 RQ3 指纹从定性升格为**定量**评测。
- **决策**：
  ① 升级采取"条件机制优先"：P1a 空间物理条件图（H/方位角/CoC 逐像素图，
    替代 D20 滑窗）+ P1b PSF 描述子图注入（与 D11 解耦签名同一套统计量）
    一次重训实施；P2 骨干（NAFNet 块/CoC 注意力/容量）与 P3（alpha 辅助头/

  ③ 数据集策略：下载 RealBokeh 3MP（真实微调+评测）、VABD（光圈连续控制评测）、

  ④ 实施时点：50k COCO 训练（v1 基线）核图之后——P1 改输入通道必须重训，

## D22 — 找补现象归因修正与三联对策（细薄前景 + 边界带门控 + P1 一并实施）

- **日期**：2026-06-12
- **背景**：50k COCO 正式训练完成（合成边界 L1 −30%，0.0521→0.0364），但 e2e
  核图（outputs/e2e_test/）发现：①真实照片上被正确虚化的前景树枝/叶被细化网
  重新锐化（"找补"，10k 占位权重时已见）；②mask 均值 0.26（训练时 0.06 的 4 倍），
  大面积点亮在非边界纹理区。**换 118k COCO 背景没有解决 → D20 时期"占位背景池
  过拟合"的归因不成立。**
- **新归因**：
  ① 合成前景全是光滑 blob（_random_blob_alpha），**没有细薄多边界结构**——
    真实细枝对网络而言"处处是边界"，按训练学到的边界策略从全焦输入抄回锐利纹理；
  ② mask 监督只教了"哪里该亮"（m_gt=clamp(10·|B_phys−GT|)），没有任何机制
    约束它在真实纹理上的误触发——合成域里平坦区 phys≈GT，网络从未见过
    "纹理丰富但物理渲染正确"的负样本形态。
- **决策（三联对策，随 P1 一次重训，不单独烧训练）**：
  ① **细薄前景**（data/synth.py `_random_thin_alpha`）：程序化树枝/草丝/栅栏
    （宽 1–6px，cv2 折线/坐标场软方波），p_thin_fg=0.5 与 blob 混采——
    "细薄物保持虚化"由精确 GT 直接监督。副产物：1–2px 细丝 alpha<0.5 不进
    视差 GT，天然模拟 DA V2 漏检细结构的真实失败模式；
  ② **边界带门控**（refine/conditioning.boundary_band）：m ← m·band，
    band = 视差边缘（5×5 极差>0.04）扩张"图内最大 |CoC|+8px"（渗色传播距离的
    物理上限）再软化。带外神经分支**结构性失效**——红线（network 不得画风格）
    从损失约束升格为结构约束。mask 监督目标同乘 band（带外目标不可优化，
    徒增噪声）。代价（接受）：视差完全漏检的细丝带外无法修复——那属于
    "物理不可知"范畴，宁可保留物理结果也不开找补的口子（P3b 的领域）；
  ③ P1b 描述子图本身对症：网络确知"此处核半径 30px"，条件上抑制锐化倾向。
- **P1 实施细节中的新决策**：
  - 取向通道不用裸 sin2φ/cos2φ，改用**各向异性张量分量** (sxx−syy, 2sxy)/trace
    ——幅度=拉伸强度、盘各向同性时自动→0，规避"圆盘取向无定义"的噪声；
    按像素方位角旋 2·az（轴量 mod π，跨画幅中心 ±π 翻转无感）。
  - 描述子表走**单色** PSF（与 D11 解耦签名同口径，1 FFT/项 vs rgb_psf 的 9）。
  - H 子带 GT：块内 H 跨度 <0.12 单带（条件图同步退化为常数，防"条件说变、
    GT 没变"自相矛盾）、<0.30 两带、否则三带；子带混合的是**渲染结果图**，
    不是 PSF（D9 禁区不触）。输入侧 render_field_patch 用同一组 (centers, weights)。
  - 虚拟画幅几何用**精确极坐标**（非径向线性近似），跨中心裁块也正确。
- **代价实测**：参数 684k→719k（+35k）；训练 0.42→0.86s/it（子带 GT 为主），
  50k ≈ 12h；显存 5.8GB 不变。e2e 整图单前向 0.13s/张@1024×1280（滑窗退役）。
- **可逆性**：configs/base.yaml refine 节三开关（spatial_cond/psf_desc/
  mask_edge_gate）全关 = 复现 v1；ckpt 存 extra_cond_ch 标记，e2e 自动选路径。

## D23 — 找补根因坐实：背景细薄层（数据修正）+ 焦邻近门（门控强化）

- **日期**：2026-06-15
- **背景**：D22 三联对策随 P1 重训完成（train_run3，50k COCO 118k）。训练指标
  健康——在更难分布（细薄前景使 phys 边界 L1 0.0521→0.0667）上净收益反增
  （−0.0157→−0.0216，相对降幅 30%→32%），loss 平稳无塌缩。但 e2e 真实图
  （21.jpg，雪地密集树枝背景）核图：**找补减轻但未根除**——zoom 对比 v1 vs P1，
  背景竹叶/树枝在 refined 列仍被部分锐化（P1 锐度明显轻于 v1，但未消失）；
  mask 均值 0.25（v1 0.26，几乎持平）。
- **诊断（band_diag.png 实证）**：
  ① **门控对密集背景根本性失效**：boundary_band 设计前提是"边界稀疏→扩张成窄带"。
    密集树枝背景**处处是视差跳变**，edge≈1，扩张图内最大 CoC（60px）后整片背景
    连成白；band 均值 0.806（≈全图）、主体本体反被挖空——与应有的"贴边界窄带"
    正好相反。门控物理上无法区分"主体遮挡边界（该修）"与"背景树枝间边界
    （不该修）"，二者都是视差跳变。
  ② **根因在训练分布的泛化方向**：D22 的细薄前景**只放在近前景**，对焦前景时
    GT 清晰 → 网络学成"细薄结构=该锐化"。真实图里细薄结构恰是**该重度虚化的
    背景树枝**——泛化方向正好相反，细薄前景对策因此部分帮了倒忙（强化"细薄→锐化"）。
- **决策（两路并举，用户拍板"两者都做"）**：
  ① **背景细薄层（数据根因修正，主菜）**：data/synth.py 新增 p_thin_bg=0.4，
    在背景侧（视差 ≤ bg_disp，更远）插入 1–2 个细薄层（复用 _random_thin_alpha），
    并**强制对焦前景**（focus_on_fg=True，需 n_fg≥1）→ 这些层重度虚化、GT 也虚化。
    网络由此拿到"细薄结构在背景深处该保持虚化"的精确监督，直接纠正泛化偏差。
    层合成顺序无忧：composite_blurred_layers 内部按视差排序，遮挡天然正确。
    验证（bg_thin_check.png）：背景细薄结构在全焦输入清晰、在 GT 重度虚化 ✓。
  ② **焦邻近门（门控强化，兜底）**：boundary_band 增 focus_gate（默认开）——
    边缘还须落在"邻域内存在清晰内容（|CoC| < focus_frac·图内最大|CoC|）"处才放行，
    远背景碎边缘（两侧都重度虚化）→ 焦门=0 → 强制回落 B_phys。物理依据：找补的
    害处全在"远离焦区、|CoC| 很大"的深背景。**坦诚局限**：对 21.jpg 这类"主体被
    密集细结构背景包围"的图，焦门只把 band 0.806→0.755（树枝紧贴主体、tol 宽使
    焦区占 43%，排不掉）——门控救不了场，真正的解是 ①。焦门对边界稀疏的图仍有效，
    保留作兜底。
- **训练侧一致性**：focus_gate 默认开 → 训练 band 同口径。对背景细薄层样本，焦门
  把远背景排除出 band → 该处 m 监督向 0 → 用 B_phys（虚化）。数据修正与门控方向
  一致、互相强化。
- **重训**：train_run4（50k，COCO 118k，同 P1 架构），验收主靶仍是 21.jpg 找补。
- **可逆性**：p_thin_bg=0 关掉背景细薄层；boundary_band(focus_gate=False) 退回
  D22 纯边缘带。

## D24 — 路线压力测试：否决端到端生成 + 开源数据角色边界 + 几何修正型网络（Plan B）

- **日期**：2026-06-15
- **背景**：train_run3 找补未根除，提出战略质疑——"能否用全新网络替代细化网？能否用
  开源（全焦图-虚化图）数据集训练？还是坚持原路线？"这是对项目核心红线（§1
  physics→network）的一次正面压力测试，结论须落盘以防日后反复。
- **先校准任务定义**：细化网的目标**不是**"生成虚化背景"，而是**修复物理渲染器在
  深度边界的瑕疵**（渗色/硬边/半透明圈，根源=DA V2 深度不准 + 分层遮挡近似）。
  风格（盘形/亮边/猫眼/旋焦/色差）100% 由物理渲染器产出，网络一个像素的风格都不画。
- **决策①：否决"端到端生成虚化网络"（用开源全焦-虚化数据直接训）**。开源数据
  （EBB!/RealBokeh/BokehMe 合成集）的虚化是某支固定镜头/固定渲染的风格，网络学会
  生成虚化 = 风格来自网络权重而非可微物理 → RQ1 可控性、RQ3 指纹、差异化定位
  三支柱全塌，等于退回 BokehMe 老路。**这是红线的直接抵触，永久否决。**
- **决策②：开源（全焦,虚化）数据不能做"边界修复 GT"**。核心论证：边界过渡的
  "正确答案"取决于焦外盘形状——**同一条边界几何，在肥皂泡/奶油/猫眼等不同像差
  系数下正确的过渡完全不同**。开源数据每张只含某支固定镜头的过渡，只能教那一支；
  我们要的是"任意像差系数下的边界过渡"，该 GT **只能由自有物理渲染器合成**
  （这正是 CLAUDE.md §7 死守在线合成、精确 alpha 分层 GT 的根本原因：保证
  "边界 GT 风格 = 渲染器风格"自洽）。用开源真实虚化当边界 GT = 用 Canon 的答案
  批改我们物理系数的卷子。
- **决策③：开源数据集的正确角色（不变，NETWORK_DESIGN §8）**——真实感微调（M4，
  只动细化网+风格保持回归）、真实验证集（评测）、镜头指纹标定素材。消除 sim-to-real
  gap 的正路是"把合成分布改得更像真实"（D23 背景细薄层正是此举），而非引入真实
  虚化当 GT。
- **决策④：坚持原路线，但确立 Plan B**。找补是**训练分布+监督设计**问题（run3 已比
  v1 明显减轻，根因诊断清晰、D23 对症），非路线死症，**不在 run4 结果出来前换路线**。
  若 run4 仍残留找补，启用 **Plan B = 几何修正型网络**（非换路线，是换网络的"修复
  方式"）：
  - **找补的结构性根源**：当前 B_neural 是网络**直接输出的颜色图** → 网络物理上有
    能力把全焦纹理画回去，找补永远是结构性诱惑。
  - **Plan B**：网络输出从"画颜色"改为"**预测几何**"（边界 alpha matte / 深度修正，
    纯几何量）→ 物理渲染器拿修正后的几何**重新渲染** → 网络从未接触颜色，
    **物理上无法找补纹理**。这是 §1 红线最干净的实现，呼应 §6.2 哲学
    "物理管一切可知的，网络只管不可知的"，且 matte GT 在合成管线免费（§6.1 P3a）。
  - 定位：介于 P3a（matte 仅辅助头）与 P3b（MPIB inpainting）之间的针对性方案，
    作为 run4 的"保险"。详细结构待 run4 后定（见 NETWORK_DESIGN §6.3）。
- **可逆性**：本条是方向确认，不改任何已实现代码；Plan B 未实施前 CLAUDE.md 架构图
  （融合公式）不动。

## D26 — PSF 裁剪/halo 尺寸须按【完整像差外缘】估，而非仅离焦

- **日期**：2026-06-18
- **背景**：复核物理渲染器时发现 `render/renderer.py` 的 `_layer_psf` 在自适应裁剪
  PSF 时，半径只用了离焦项 `4·|W020|`，**完全漏算球差 W040 与视场项**（彗差/像散/场曲）。
- **症状（实测）**：近焦面的强球差层（soap_bubble/cream 预设 W040=±1.5~2，离焦 W020≈0）
  PSF 被裁掉外缘 **最高 27% 能量**；被裁掉的恰是【盘边缘的肥皂泡亮环】——本项目的
  标志特征。而 `pupil_to_psf` 裁剪后会把 PSF 重归一化(sum=1)，把这截断**悄悄掩盖**，
  能量检查/肉眼都难发现。`render_tiled` 的 tile `halo` 同病：强彗差甩尾在拼接处被
  邻域不足截断 → 接缝。
- **决策**：新增 `psf_extent_px(coeffs, w020_total, H)`，按各像差项在光瞳边缘(ρ=1)的
  最大光线斜率累加估外缘半径（与 `optics/calibrate.py` 同一套 `r_px = 2·dW/dρ|_{ρ=1}`）：
  `extent ≈ 4|W020| + 8|W040| + 6·H·|W131| + 4·H²·(|W222|+|W220|) + 余量`。
  `_layer_psf` 裁剪与 `render_tiled` halo 均改用它。
- **理由**：裁剪/halo 是离散窗口选择，宁可保守取大（仅多花一点 FFT），绝不能截断
  PSF 形状——尤其球差亮环是焦外风格化的核心可控特征，截断会直接污染可控性监督与指标。
- **验证**：漏裁能量 **27% → ≤0.1%**（全预设含彗差/像散高像高）；6 预设×{global,tile}
  在"guard 提为 error"下零告警、输出有限；对 W040/K 梯度正常（指纹标定链路不破）。
  视觉 `outputs/render_test/soap_crop_fix.png`。
- **影响**：纯修复，物理公式与符号约定不变，向后兼容（旧结果只是边缘被截，新结果更完整）。
  全程 detach，不进梯度图。
- **可逆性**：高——半径公式集中在单个函数，可调系数。

## D27 — 高光权重重分配 WR：把"高斯波"高光掰成"方波"（接 BokehMe++ Fig.3）

- **日期**：2026-06-18
- **背景**：BokehMe++ 指出，8-bit/有限动态范围的全焦原图里，过曝高光在拍摄时已能量
  溢出 + 带柔光晕——看起来比真实物理尺寸大、边缘已是渐变。直接拿去和散景核卷积，
  衰减叠加放大 → 弥散圆边缘像高斯般缓慢衰减，光斑扁平糊成一团，缺乏单反晶莹层次
  （Fig.3 的 w/o WR 反例）。这是【焦外风格化的要点】。
- **现状诊断**：上一版 `srgb_to_linear` 只做【乘性提亮】`lin·(1+gain·w)`：高光变亮了，
  但内部仍是带峰的渐变、边缘仍软（实测中心剖面 0.37→9→0.37）——正是"高斯波"，
  **没有解决** WR 要解决的问题。
- **决策**：新增 `redistribute_highlights`（在散射前、线性域执行），按原文三步把高光
  能量分布从高斯波掰成方波：① sigmoid 锐利掩膜(温度 `highlight_softness`)定 footprint
  → ~1px 陡边；② 用各像素【自身色度 × 统一平顶亮度 peak=1+gain】替换原值 → 内部均匀
  平顶（压缩数值范围）、保留红/蓝色相；③ 内部取平顶、外部取原值，由掩膜线性混合。
  锐利掩膜天然排除去饱和柔光晕（阈值以下 m≈0）。新增 `RenderControl.highlight_softness`。
- **理由**：`PSF ⊛ 平顶方波 ≈ PSF 本身` → 弥散圆获得锐利边缘 + 均匀内部 + 红蓝叠加的
  分层折射感。这是 PSF 形状(球差亮环/猫眼/多边形)能在最终图上【被看清】的前提：
  软边高光会把 PSF 自身的边缘特征抹糊。
- **验证**：WR 后中心剖面 0.45→9→0.45 平顶 + 2px 锐边（对比旧 0.37→9→0.37 尖峰）；
  Fig.3 复现 `outputs/render_test/wr_fig3.png`（红/蓝高光 w/ WR 边缘更锐、内部更匀、
  叠加分层更清）；全预设×双路径×WR 开启零告警、有限；对 image/highlight_gain/W040/K
  全可微（指纹标定不破）。
- **影响**：替换原乘性提亮逻辑（`highlight_gain`/`highlight_thresh` 字段语义保留）。
  默认 `highlight_gain=0` 时完全不触发，非高光场景/训练可关。
- **可逆性**：高——逻辑集中在 `redistribute_highlights` 单函数；`highlight_gain=0` 即旁路。

## D28 — 渲染器真实性复核：WR 彩色高光检测(通道最大值) + 光瞳 apodization 抑制相干衍射边环

- **日期**：2026-06-18
- **背景**：肉眼复核 `demo_lights_grid.png` 发现两处与真实光学冲突，对照 BokehMe++ 诊断。
- **问题 1：WR 漏检彩色高光 → 彩色弥散圆显著偏暗、半成形不自然亮边**
  - 根因：`redistribute_highlights` 用【亮度均值】判高光阈值。但过曝是【逐通道】饱和的——
    一支饱和红 (R≈1, G≈B≈0.15) 亮度均值仅 ~0.34，低于阈值 → 掩膜 m≈0.11 → WR 几乎不
    触发，彩色光斑停留在原始柔糊态（旧乘性提亮同病）。白光斑被充分提亮、彩色光斑没有，
    叠加显示归一化后彩色显得更暗，且 11% 部分 WR 留下半成形的怪异亮边。
  - 决策：改用【通道最大值 detect=lin.amax(0)】判阈值（任一通道饱和即判高光），
    平顶上色改 `chroma=lin/detect`（饱和通道顶到 peak、其余按比例，保色相、避免按亮度
    归一时彩色被过度放大）。红/蓝光斑与白光斑一视同仁地变亮、锐化。
- **问题 2：理想镜头(无像差)散景盘出现不自然亮边 → 误以为"像差没解耦"**
  - 根因：`|FFT{硬边光瞳}|²` 是【相干】仿真，在盘的几何边界处产生一圈高对比【衍射亮边环】
    （理想盘实测边缘亮度 1.25× 内部）。真实相机是宽光谱+部分相干+点源有限角直径+叶片
    非理想硬边，这圈相干环被抹掉。**这不是像差耦合**——PSF 层面 cream→亮心(0.59)、
    soap→亮环(2.4×)、ideal→平(无环) 本就分离良好（decoupling 矩阵对角占优）；是相干
    衍射伪环让"理想"看着像有 soap 味的亮边。
  - 决策：对光瞳边缘做轻度 **apodization**——把默认 `softness` 0.02→0.04（新增模块常量
    `pupil.DEFAULT_SOFTNESS`），等效"部分相干/有限源尺寸"的物理平滑。理想盘边环 1.25×→1.10×，
    而球差亮环/猫眼/多边形等像差特征几乎不变（它们不在那一圈）。光谱平均(SPECTRAL_SAMPLES)
    负责抹【盘内】菲涅耳环、apodization 负责抹【盘边】相干环，分工互补。
  - 取舍：softness 太大→盘边过糊、多边形角变圆；太小→相干环回来。0.04 是折中。
    试过加密光谱采样(3→5)对盘内 ripple 无稳定增益却多 67% FFT（违反训练算力预算），故不采用。
  - 配套：softness 改变后重跑 `optics.calibrate`，斜率 3.93→3.87（仍贴近理论 4，截距吸收）。
- **验证**：6 预设×{global,tile}×WR×apodization 零告警、有限非负；端到端可微(W040/gain/K)；
  decoupling 对角元素 +1.00/+1.00/−0.90/−1.00（保持/略升）；视觉 `outputs/render_test/renderer_fixes.png`
  （ideal 中性盘、cream 亮心、soap 亮环、hex 多边形、swirl 猫眼，各风格清晰可分）。
- **影响**：纯真实性修复，物理公式/符号约定不变。WR 字段语义保留；softness 默认值上调影响
  所有 PSF（更接近真实镜头边缘），已重标定。
- **可逆性**：高——WR 检测在单函数；softness 是单个常量；光谱采样表一行。

  ① 网络预测什么几何量 → **边界 alpha matte**（非深度修正）：matte 直指病灶（找补本质是
